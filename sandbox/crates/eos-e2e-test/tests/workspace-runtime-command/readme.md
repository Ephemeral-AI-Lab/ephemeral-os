# workspace-runtime-command

## Overview

`workspace-runtime-command` owns explicit live E2E coverage for the command daemon contract across protocol smoke, core lifecycle, transcript-log recycling, ephemeral workspace process-group behavior, externally killed process groups, invisible background subprocesses, and isolated workspace command cleanup. It exercises `api.v1.exec_command`, `api.v1.write_stdin`, `api.v1.command.collect_completed`, `api.v1.command.cancel`, `api.v1.command_count`, `api.v1.cancel`, and `api.layer_metrics`. Module config lives at `crates/eos-e2e-test/tests/workspace-runtime-command/config/default.test.yml`.

## Checklist

- [ ] workspace-runtime-command-lifecycle: `exec_command`, `write_stdin`, collect-completed, cancel, timeout, command count, and timestamped transcript behavior are coherent for running and completed commands.
- [ ] workspace-runtime-command-transcript-progress: Timestamped transcript progress remains observable through read_progress and operation-scoped stdin responses.
- [ ] workspace-runtime-command-transcript-recycling: PTY transcript logs are transient capture files and are removed after finite pre-yield completion, collect-completed finalization, cancellation, external process death, parallel prompt cleanup, and isolated command cancellation.
- [ ] workspace-runtime-command-cancel-kills-group: Command cancellation tears down the owned process group, drains commands, releases leases, and leaves no descendant marker processes.
- [ ] workspace-runtime-command-teardown-control-cancel: Ctrl-C (`\u0003`) and Ctrl-D (`\u0004`) stdin payloads both route to command cancel, drain the command, and recycle the transcript log.
- [ ] workspace-runtime-command-detached-child-contract: `nohup` and `setsid nohup` descendants have an explicit tested contract instead of an accidental leak policy.
- [ ] workspace-runtime-command-ephemeral-process-group: Ephemeral workspace commands stay running until all same-pgid subprocesses exit and collect exactly one terminal result.
- [ ] workspace-runtime-command-isolated-cleanup: Isolated workspace command cancellation and exit discard mode make same-port re-entry reusable.
- [ ] workspace-runtime-command-command-matrix: At least ten shell command families run through `exec_command` with multiple variants, foreground completion, changed-path assertions, and bounded timing/resource telemetry.
- [ ] workspace-runtime-command-prompt-read-cancel: Prompt-style stdin input, read_progress tail reads, collect-completed reads, and cancellation behave coherently without replaying consumed output.
- [ ] workspace-runtime-command-long-running-stderr-stdin: Long-lived commands that emit stderr, consume stdin, and then keep sleeping remain running and uncollectable until terminal action.
- [ ] workspace-runtime-command-external-process-death: Externally signaled children and process groups finalize or become observable through `write_stdin` without leaking commands, leases, or marker processes.
- [ ] workspace-runtime-command-invisible-background-process: Silent redirected same-pgid subprocesses keep commands running even after the foreground command exits, preventing invisible background work from being missed.
- [ ] workspace-runtime-command-parallel-load: Commands execute under the configured `1/3/6/12` parallel ladder without leaked commands, leaked leases, unbounded upperdir/run-dir growth, or lost publishes.
- [ ] workspace-runtime-command-parallel-prompt-load: Prompt-style commands run at the configured `1/3/6/12` ladder with isolated stdin payloads, transcript-scoped replies, clean cancellation, and zero leaked commands or leases.
- [ ] workspace-runtime-command-negative-invalid: Nonzero exits, missing commands, unknown command ids, bogus stdin/cancel requests, and collect-completed misses return structured payloads without leaking commands or leases.
- [ ] workspace-runtime-command-output-backpressure: Large PTY output bursts stay timestamped in the transcript-backed stream and remain cancellable without leaked commands or leases.
- [ ] workspace-runtime-command-stdin-backpressure: Bounded stdin writes to non-reading consumers return promptly, leave the command cancellable, drain resources, and recycle transcript logs.
- [ ] workspace-runtime-command-isolated-prompt-parity: Isolated commands accept prompt input, support read_progress snapshots and collect-completed reads, cancel cleanly, and discard private command writes on isolated exit.
