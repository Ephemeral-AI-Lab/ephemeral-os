export async function runToolBatch( calls: ToolUseBlock[], tools: ToolRegistry, signal: AbortSignal, emit: (event: AgentEvent) => void, ): Promise<ToolResultBlock[]>

Execute one assistant message's `tool_use` batch: fully concurrent under a cap of 8, results assembled in `tool_use` order regardless of completion order. A thrown tool error or an unregistered name becomes an `is_error` result — never an exception, never a sibling cascade. On abort the batch settles immediately: completed calls keep their real results and every other block gets a synthetic `"interrupted"` error result, so the transcript never holds an unanswered `tool_use`.
