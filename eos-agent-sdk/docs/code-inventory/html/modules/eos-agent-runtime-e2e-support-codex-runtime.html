export function loadConfiguredCodexRuntime( env: NodeJS.ProcessEnv = process.env, ): ConfiguredCodexRuntime

Probe the codex coding-plan entry through the runtime's own loader (`.eos-agents/llm_clients.json` + codex CLI auth file). Loading is local - config parsing and JWT-claim validation, no network - so a missing or stale credential becomes a suite skip carrying the loader's startup error, never a test failure.
