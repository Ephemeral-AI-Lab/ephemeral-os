export interface LlmClientContractBinding

One binding of the `LlmClient` leg contract (Phase 02 §4.5): the scenarios decide how each behavior is induced (fixture-fetch client vs live profile + eliciting prompt); the assertions are shared. Strictness is data: deterministic bindings add the `exact` golden block, live bindings omit it. There is no live/unit branching inside scenario bodies.
