The dual transcript: `displayed_messages` for users, `llm_messages` for the provider. Every append writes both lists in one call (single-writer rule); the lists diverge only by declared policy — partial assistant output is displayed-only, and a future compaction phase rewrites `llm_messages` while `displayed_messages` stays append-only. `llmMessages()` is the only history source for a provider request.
