EventStream class · line 67 · public
Module 
export class EventStream implements AsyncIterable<AgentEvent>

A push-fed queue consumed as one pull-based `AsyncIterable`: - single consumer: a second `[Symbol.asyncIterator]()` call throws, - pushes never block the loop; the buffer is unbounded (the consumer is in-process; backpressure belongs to the server phase), - `close()` completes iteration once the buffer drains, - an early `break`/`return()` detaches: later pushes are discarded while the run continues; a stream nobody iterates retains every event.
