export class RunBackgroundTaskSupervisor implements BackgroundTaskSupervisor

The per-run task registry — the registry IS the open set the park/exit gates read. On `done` resolving, the supervisor does exactly one thing: invoke `onCompletion` once, awaited and bounded (or remove immediately for `silent` tasks); a task is removed the moment its completion handling finishes, and every removal wakes a parked loop. The gate methods (`isEmpty`/`count`/`changeCount`/`waitForChange`/`disposeAll`) are engine-facing internals, deliberately absent from the public `BackgroundTaskSupervisor` interface.
