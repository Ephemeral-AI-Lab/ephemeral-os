bindTerminalTool function · line 46 · public
Module 
export function bindTerminalTool<T>( fn: AgentOutcomeBinding<T>, scope: RunScope, gate: TerminalGate, ): TerminalBinding<T>

Bind the minted terminal contract as an ordinary-looking tool whose successful result ends the run. Order per call: schema -> gate -> pre hooks -> post hooks (on the provisional result) -> `onSubmit`. Hooks run BEFORE the handler so a hook deny — like a handler reject — costs the host nothing; once `onSubmit` accepts, nothing can stop the finish, so the accepted value and host state cannot diverge (single mutator).
