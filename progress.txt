and `iteration/manager.py:_evaluator_pass_summary_for` only read the free-text summary. Tier B
is a rendering-only plumbing fix — no Attempt/Iteration schema, no Alembic, no audit-contract
field changes.

### Codebase landmarks (for future iterations)
- The shared test scaffolding lives at `backend/tests/unit_test/test_engine/_fake_provider.py` (FakeProviderClient) and `_retry_test_support.py` (ScriptedRetryAgent + tool fixtures + real-loop agent builder). Re-use for any future retry / engine-layer test that wants real-loop coverage without a live provider.
- The static guard `test_launcher_runner_kwargs_do_not_disable_engine_retry` (in `test_attempt_launcher_retry.py`) locks the invariant that `EphemeralAttemptAgentLauncher._run_launch` does NOT pass `max_terminal_retries` — the engine's default of 1 retry is the contract. Companion to `test_runner_kwargs_contract.py`'s frozen-kwarg set.
- RESOURCE_LIMIT post-dispatch with `terminal_tools` non-empty requires pre-seeding `tool_calls_used = tool_call_limit`. The reserved-slot rule (`_consume_tool_budget_or_reject` in `tools/_framework/execution/tool_call.py`) blocks non-terminal calls at `tool_calls_used == limit - 1`, so non-terminals alone can never push the counter to limit. This is the structural quirk that any future real-loop RESOURCE_LIMIT test must work around.
- For helper tools (`ask_advisor`, `ask_resolver`, `run_subagent`), monkeypatching must hit BOTH `engine.api.run_ephemeral_agent` and `engine.agent.lifecycle.run_ephemeral_agent` because `engine.api` is a lazy proxy that may or may not have cached the canonical reference yet at call time.
- The `tools.ask_helper.{ask_advisor,ask_resolver}` import path is ambiguous: the submodule and the FunctionTool share the name. Use `sys.modules["tools.ask_helper.ask_advisor"]` for module-level monkeypatching of compose_helper_bundle.
