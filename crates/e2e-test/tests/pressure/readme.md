# pressure

## Overview

`pressure` owns cross-subsystem stress coverage for live `eosd`: concurrent direct file ops, reads, ephemeral exec, LayerStack auto-squash, OCC disjoint publish, command cancellation, plugin refresh, isolated handle pressure, post-cancel daemon readiness, and JSON resource reports. It exercises daemon ops including `sandbox.file.write`, `sandbox.file.read`, `sandbox.command.exec`, `sandbox.command.cancel`, `sandbox.command.count`, `sandbox.checkpoint.layer_metrics`, `sandbox.runtime.ready`, `sandbox.plugin.ensure`, `sandbox.plugin.status`, `plugin.generic.query`, and `sandbox.isolation.*`. Module config: `crates/e2e-test/tests/pressure/config/default.test.yml`. This is one unified E2E contract. Configured pressure concurrency is fixed at level `12`; the runner no longer exposes fast/full/heavy pressure lanes.

## Checklist

- [ ] pressure-ladder-file: Direct file write/read pressure passes at concurrency `12`, publishes coherent content, and does not leak active leases.
- [ ] pressure-ladder-exec: Ephemeral exec pressure passes at concurrency `12`, publishes changed workspace files, and releases overlay leases.
- [ ] pressure-ladder-command: Command sessions start, cancel, and drain at concurrency `12` with session count and active leases returning to zero.
- [ ] pressure-ladder-occ: OCC disjoint writes and same-path conflict pressure return coherent payloads at concurrency `12`.
- [ ] pressure-ladder-plugin: Plugin refresh/dispatch pressure remains coherent at configured levels and keeps refresh/process counts bounded.
- [ ] pressure-isolated-cap: Isolated handle pressure either runs under a high-cap config or asserts configured cap rejection above the default limit.
- [ ] pressure-squash-bound: Repeated overwrite pressure keeps manifest depth under the operational auto-squash target while preserving latest content.
- [ ] pressure-recovery-cleanup: Midflight cancel and cancel bursts leave daemon readiness intact and drain commands, active leases, and marker work.
- [ ] pressure-resource-report: E2E runs emit JSON summaries from response meta, trace_resources, and leak counters before strict regression thresholds are introduced.
- [ ] pressure-bundled-sandbox-capstone: A protocol-only mini-bundle combines scaffold, disjoint writes, same-path conflict, squash, background command, isolated private write, commit projection, and plugin dispatch without agent-loop assertions.
- [ ] pressure-sandbox-only-boundary: Pressure capstone coverage remains limited to sandbox protocol and trace/resource behavior, leaving TaskCenter state, advisor gates, and notification-loop assertions to agent-core tests.
- [ ] pressure-overlay-disk-o1: Concurrent overlay execs over a shared lowerdir base keep each per-op upperdir delta-sized and do not multiply LayerStack storage, proving O(1) overlay disk with respect to the number of parallel readers/writers.
- [ ] pressure-resource-memory: Resource samples carry cgroup memory.current/peak and process RSS gauges, and a large-base overlay op keeps daemon memory within a bounded ceiling (a loose regression gauge inflated by page cache, not a tight O(1) bound).
- [ ] pressure-multi-caller: N distinct agents (distinct `caller_id`s) concurrently write disjoint and same-path files on one shared LayerStack; every disjoint write commits and is readable across callers, same-path races resolve to one whole winner, and per-caller leases drain to zero.
- [ ] pressure-concurrent-squash: A driver thread crosses the auto-squash depth target while a concurrent pool writes disjoint paths, so auto-squash runs during live publishes; manifest depth stays bounded, no superseded layer dirs are stranded, no data is lost, and no leases leak.
- [ ] pressure-cross-mode-public-isolated: Public and isolated callers race same-path writes at concurrency `12`; isolated callers keep private content while open, public content survives after isolated exit, and all leases drain.
- [ ] pressure-soak: A mixed file/exec/command-cancel round over a fixed working set, repeated many times, keeps the leak counters (active leases and commands) at zero every round, auto-squash bounds manifest depth and reclaims storage as a sawtooth so storage does not creep with round count, and the daemon stays ready throughout.
