# workspace-runtime-isolated

## Overview

This module owns the unified live E2E contract for isolated-session lifecycle, private upperdir routing, SetNs namespace teardown, network isolation, file-tool routing, and live isolated remount compaction while a caller is isolated. It exercises daemon ops `sandbox.isolation.enter`, `sandbox.isolation.status`, `sandbox.isolation.exit`, `sandbox.isolation.test_compact_remount`, `sandbox.file.read`, `sandbox.file.write`, `sandbox.file.edit`, `sandbox.command.exec`, and `sandbox.command.cancel`. Module config lives at `crates/e2e-test/tests/workspace-runtime-isolated/config/default.test.yml`.

## Checklist

- [ ] isolated_network-lifecycle-pin: Enter and status expose the same manifest pin, and exit unregisters the handle, releases the lease, and reports holder/cgroup teardown.
- [ ] isolated_network-private-persistence: Isolated writes and reads persist across tool calls while the handle is open and report isolated-network workspace/mutation fields.
- [ ] isolated_network-no-publish: Isolated write, edit, and exec paths do not publish through OCC or release public LayerStack leases.
- [ ] isolated_network-discard-exit: Exit evicts the private upperdir, and public or host reads cannot see isolated-only files afterward.
- [ ] isolated_network-network-isolation: A dedicated isolated netns and veth allow an isolated caller to bind a port already held in the container netns.
- [ ] isolated_network-same-netns-conflict: Reusing a port in the same namespace conflicts unless the isolated session has exited and cleaned the prior command.
- [ ] isolated_network-exit-cleanup: Exit or cancel drains long-lived isolated commands and removes namespace resources so re-enter starts cleanly.
- [ ] isolated_network-tool-routing-contract: File tools in isolated mode return explicit workspace, mode, mutation, publish, and conflict fields, then route back to host after exit.
- [ ] isolated_network-exec-private-discard: Exec writes inside isolated mode remain private while open, then disappear on exit without OCC publication.
- [ ] isolated_network-lifecycle-gates: Enter rejects active sandbox-bound background work, repeated enter reports an already-open or lifecycle-in-progress state, and exit drain timeout/retry remains stable.
- [ ] isolated_network-network-peer-block: Isolated callers can reach their own loopback service but cannot reach peer isolated sessions through namespace or bridge addresses.
- [ ] isolated_network-audit-trail: Enter, exit, and isolated tool calls keep teardown trace events, discarded-byte facts, and no-publish behavior observable through response meta and trace-store rows.
- [ ] isolated_network-ttl-eviction: Short test config TTL evicts idle isolated sessions and preserves coherent status/list-open behavior.
- [ ] isolated_network-cross-mode-consistency: Public writes from non-isolated callers can advance OCC while an isolated caller is open, but the isolated pin/private upperdir stays stable and is discarded without overwriting public state.
- [ ] isolated_network-compact-remount: An open isolated session can compact/remount onto a bounded lowerdir set while preserving public snapshot and private upperdir state, and non-open host callers report `not_open`.
- [ ] isolated_network-daemon-restart-reconciliation: A hard daemon restart runs startup orphan reconciliation, reaping an open isolated handle so `list_open` empties, releasing its snapshot lease with none stranded, and serving fresh isolated enters again.
