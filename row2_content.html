PR 10042

Tweaks logic in apply_filters, and improves test coverage for filtering on columns with missing min/max statistics.

Closes #9975. Since the update to fastparquet 2023.2.0, the test suite fails with the following error:

TypeError: An error occurred while calling the read_parquet method registered to the pandas backend.
Original Message: '<=' not supported between instances of 'NoneType' and 'str'

The failing case is test_select_filtered_column[fastparquet]. The root cause is in apply_conjunction within apply_filters:

def apply_conjunction(parts, statistics, conjunction):
    for column, operator, value in conjunction:
        if operator == "in" and not isinstance(value, (list, set, tuple)):
            raise TypeError("Value of 'in' filter must be a list, set, or tuple.")
        out_parts = []
        out_statistics = []
        for part, stats in zip(parts, statistics):
            if "filter" in stats and stats["filter"]:
                continue  # Filtered by engine
            try:
                c = toolz.groupby("name", stats["columns"])[column][0]
                min = c["min"]
                max = c["max"]
            except KeyError:
                out_parts.append(part)
                out_statistics.append(stats)
            else:
                if (
                    operator in ("==", "=")
                    and min <= value <= max
                    or operator == "!="
                    and (min != value or max != value)
                    or operator == "<"
                    and min < value
                    or operator == "<="
                    and min <= value
                    or operator == ">"
                    and max > value
                    or operator == ">="
                    and max >= value
                    or operator == "in"
                    and any(min <= item <= max for item in value)
                ):
                    out_parts.append(part)
                    out_statistics.append(stats)
        parts, statistics = out_parts, out_statistics
    return parts, statistics

When min and max are None (missing statistics), comparisons fail.

PR 10120

Sibling to dask/distributed#7708. Closes #10118.

Problem

DataFrame.categorize drops all annotations from the input graph when computing the categories even if low level fusion is deactivated.

Example

from __future__ import annotations
import dask
import dask.dataframe as dd
from dask.distributed import Client, LocalCluster

def foo():
    cluster = LocalCluster()
    client = Client(cluster)
    df = dask.datasets.timeseries(
        start="2000-01-01",
        end="2000-03-01",
        dtypes={"x": float, "y": str},
        freq="10 s",
    )
    df = dd.shuffle.shuffle(df, "x", shuffle="p2p")
    df = df.categorize(columns=["y"])
    df.compute()

if __name__ == "__main__":
    foo()

Where do we drop annotations?

Within categorize, we compute the categories. In this step, we convert from a high-level graph to a low-level one without preserving the annotations:

dsk.update(df.dask)
# Compute the categories
categories, index = compute_as_if_collection(
    df.__class__, dsk, (prefix, 0), **kwargs
)
