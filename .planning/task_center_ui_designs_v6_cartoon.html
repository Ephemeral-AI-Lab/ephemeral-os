Anatomy at a glance 🧩
Five layers. Each maps to a folder. Each carries its own JSON.
🗂Run
The whole show

One audit folder per run. Holds metrics + perf report + sandbox events.

run.json
🎯Goal
What was asked for

Owns ordered iteration_ids. Spawned by a task (entry, or a nested handoff).

goal_NN_<uuid>/
📐Iteration
A segment of work

One go at the goal. May close with a continuation_goal → next iteration.

iteration_NN_<uuid>/
🎲Attempt
One full try

Planner ▸ Executor(s) ▸ Evaluator. If fail and budget left → next attempt.

attempt_NN_<uuid>/
🧙Task
One agent's turn

Three per attempt: planner, executor (=gen), evaluator. Carries spec + spawn_reason + summaries.

0N_role_<uuid>:role/

Two event streams 🎙
Each task has its own message.jsonl. The run root has sandbox_events.jsonl. The UI's event log should merge them; the disk keeps them separate.

Nested goals: flat on disk, deep in spirit 🪆
When an executor calls submit_execution_handoff, a new goal_NN/ dir is created — next to its parent, not inside. The recursion is computed via requested_by_task_id walking, by nested_goal_depth().
🎯Goal 1
goal_01_cd4c…
requested_by: :entry
2 iter · 4 att · "audit recorder"
🎯Goal 2 🌀nested
goal_02_a1b2…
requested_by: 02_executor (A2.2 / exec_b)
depth=2 · 1 iter · 1 att · "diagnose flake"
🎯Goal 3
goal_03_e9f0…
requested_by: 02_executor (Goal 2's exec)
depth=3 · queued · "minimal repro"
↪ on disk: three sibling folders. in the UI: Goal 2 nests under Goal 1, Goal 3 nests under Goal 2 — derived by walking requested_by_task_id.
