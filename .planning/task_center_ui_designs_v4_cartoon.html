Meet the model 🎬
Three nested boxes, one rule each. That's it.
1

🎯Goal

The thing the user asked for. Created once. May be created again by a generator (see #4 — nested goals).

2

📐Iteration

One segment of work toward the goal. Iter N+1 only fires if iter N said "I'm done locally, but the goal isn't" — i.e., a continuation.

3

🎲Attempt

One full try inside an iteration. If it fails, you retry: Attempt N+1 fires from the same iteration. Capped by attempt_budget.

The iteration chain 🧗
When an iteration closes with a continuation_goal, the next iteration picks up where it left off.
📐Iter 1 closed · "I did my bit"
A 1.1 A 1.2
summary: "removed buffer in recorder.py; tests green"
continuation
📐Iter 2 running · "okay, the rest"
A 2.1 A 2.2 ◀
goal: "remove buffer call-sites in stores.py"
↳ If iter 2 also says "still more to do" → iter 3 will appear. (Budget caps the chain.)
The Nested-Goal Portal 🌀
A generator can call submit_execution_handoff(goal) mid-attempt and spawn a whole new Goal inside itself. Yes — Goal → Iter → Att inside Goal → Iter → Att. Turtles.
