Merged architecture wiki for EphemeralOS modules. Use the module layer for broad ownership, the page layer for local concepts, and cross-module links for real runtime edges.

Modules §

Sandbox WorkspaceLayerStack, overlay, OCC, daemon routing, providers, plugins, and workspace execution modes. ToolsAgent-facing tool definitions, framework lifecycle, terminals, helpers, background tasks, and TaskCenter submission surfaces.

Cross-Module Map §

tools to sandboxSandbox tools call typed sandbox APIs whose runtime semantics are explained by Sandbox Workspace.
tools to daemonBackground tools, subagents, and helper tools interact with engine and sandbox daemon lifecycles documented in Daemon and Background Tasks.
private scratchIsolated workspace tools are the agent-facing entry points for the private scratch lifecycle detailed in isolated_workspace.

Maintenance §

  • Shared assets live in docs/architecture/assets.
  • Module pages live in module folders; future modules should add a sibling directory and global search entries.
  • Cross-module links are allowed when they express a real code path, ownership edge, workflow dependency, or diagnostic route.
