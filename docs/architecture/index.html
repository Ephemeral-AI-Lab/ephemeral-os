Merged architecture wiki for EphemeralOS modules. Use the module layer for broad ownership, the page layer for local concepts, and cross-module links for real runtime edges.

Modules §

Sandbox WorkspaceLayerStack, overlay, OCC, daemon routing, providers, plugins, and workspace execution modes. ToolsAgent-facing tool definitions, framework lifecycle, terminals, helpers, background tasks, and TaskCenter submission surfaces. Agent LoopsEngine run lifecycle, model-provider streaming, prompt/context assembly, background operations, sandbox-provider bridges, notifications, and message artifacts. TaskCenterGoal -> Iteration -> Attempt state model, planner-generator-evaluator harness, context engine, role profiles, and terminal state submissions.

Cross-Module Map §

tools to sandboxSandbox tools call typed sandbox APIs whose runtime semantics are explained by Sandbox Workspace.
tools to daemonBackground tools, subagents, and helper tools interact with engine and sandbox daemon lifecycles documented in Daemon and Background Tasks.
private scratchIsolated workspace tools are the agent-facing entry points for the private scratch lifecycle detailed in isolated_workspace.
agent loop to toolsMain agent loop defers, dispatches, and terminates tool calls whose framework contracts are documented in Tools Framework.
agent loop to model providersModel Provider Bridge follows backend/src/providers active model resolution, ApiMessageRequest, stream event translation, auth, retry, and provider errors.
agent loop to sandboxProvider Sandbox Bridge follows sandbox-backed tools through public API transport, daemon routing, provider adapters, OCC, overlay, and isolated workspace gates.
agent loop to TaskCenterPrompt and Context shows how TaskCenter profiles, context recipes, terminal routing, task guidance, and skill rows become launch-time prompt rows.
TaskCenter to toolsTaskCenter Terminal Tools explains how planner, executor, verifier, evaluator, advisor, resolver, and explorer submissions enter the tool framework and engine terminal stop path.
TaskCenter to sandboxTaskCenter Bridges traces run bootstrap, sandbox binding, agent launch metadata, workspace tools, and live harness artifacts across engine, tools, and sandbox modules.
