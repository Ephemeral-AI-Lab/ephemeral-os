Prompt assembly is deliberately split. The engine builds the system prompt from settings and the loaded profile body. TaskCenter launch code then adds rendered role context, task guidance, and an optional skill row. The old packet, recipe registry, context outline, and token-priority renderer path has been removed.

Runtime And Profiles #

build_runtime_system_prompt() reads settings and returns the base runtime instruction text, including fast-mode guidance when configured. engine.agent.factory._build_agent_system_prompt() appends the loaded AgentDefinition.system_prompt and the terminal warning section derived from the actual tool registry.

Main profile loading is file-backed. load_agents_tree() parses Markdown frontmatter, defaults the name from the filename, validates relative skill and terminal-routing paths, and prepends _main_role_contract.md to the main-role profiles. The profile body remains the system-prompt body; launch-specific context is not injected here.

Frontmatter
Main profiles declare role, allowed tools, terminals, notification triggers, context role id, optional skill, and optional terminal-routing module.
Terminal warning
_finalize_tool_registry_and_prompt() derives terminal names from registered tool classes and appends the termination-condition warning.
Validation
validate_agent_definitions_resolved() runs skill-file linting. It no longer validates against a process-global context recipe registry.

Composer Rows #

AgentEntryComposer.compose() is the TaskCenter launch seam. It resolves the effective profile and terminal set, asks ContextEngine for an AgentContext, renders context XML, renders task guidance from the same context object, and appends the optional skill row.

Launch rowOwnerCurrent shape
Systemengine.agent.factoryRuntime base + loaded profile body + terminal warning section.
Contextcontext_engine.xml<context role="..."> rendered from AgentContext.
Task guidancecontext_engine.task_guidance + skill_message._wrap_task_guidance()Context outline, context limits when relevant, what-to-do directive, and <terminal_tool_selection>.
Skillbuild_skill_message()Load skill: ..., stripped skill body, and a byte-equal terminal selection block when terminals exist.

EphemeralAttemptAgentLauncher._run_launch() maps those rows into the runner call. When both task guidance and skill exist, the skill row is the runner prompt and context plus guidance are initial messages. When only guidance exists, guidance is the runner prompt and context is the initial message. With no guidance, context is the prompt.

Effective terminals Planner and executor frontmatter stays broad. Launch routing filters it by nested workflow depth: nested planners keep only submit_plan_closes_goal, and nested executors lose submit_workflow_handoff. The registered profile is not mutated.

Context Engine #

ContextScope carries launch identity: role, workflow id, iteration id, attempt id, and task id when needed. ContextEngineDeps carries workflow, iteration, attempt, and task stores. ContextEngine.build() dispatches by scope.role and returns AgentContext.

typed documentAgentContext owns role, sections, directive, and context limits.
pure renderingrender_context_xml() escapes text and attributes and renders multiline body tags. It never consumes pre-rendered XML blocks.
no registryThere is no recipe registry, packet persistence store, tag dictionary, or token-priority compression path in the current launch context code.
lifecycle boundaryContext construction reads persisted TaskCenter state. It does not decide retry, deferral, child-workflow close, or attempt closure.

Role Shapes And Guidance #

RoleContextGuidance directive
planner<workflow> with workflow goal, prior iteration outcomes, current iteration goal, and previous attempt evidence.Plan generator and reducer tasks for <current_iteration><goal>.
generator<dependencies> plus <assigned_task>. No workflow goal or attempt-wide plan.Complete <assigned_task> using <dependencies>.
reducerThe same dependency + <assigned_task> shape as generator.Complete <assigned_task> using <dependencies>.

Only execution evidence is rendered as task outcomes. Planner task outcomes are UI metadata on the planner task and are excluded from attempt, iteration, workflow, and planner-history projections.

Boundaries #

Submission owns mutationPlanner, generator, reducer, and handoff terminals mutate TaskCenter state through submission contexts and orchestrators.
Outcome projection is explicitAttempt outcomes contain generator/reducer execution outcomes; iteration outcomes are reducer successes from all attempts plus failed execution outcomes from the final failed attempt; workflow outcomes derive from the latest iteration.
Child workflow handoff is flattenedA parent generator waiting on a child workflow receives the child workflow outcomes directly when the child closes, with no wrapper task outcome.

Evidence And Refresh #

Refresh Triggers

  • Profile frontmatter, terminal routing, or terminal descriptor changes.
  • Context XML tags, AgentContext sections, or task guidance changes.
  • Outcome projection changes that alter planner/generator/reducer context evidence.
  • Attempt launcher row-ordering changes.
