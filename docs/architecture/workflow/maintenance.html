Refresh Triggers #

Workflow Iteration Attempt

Refresh when Workflow, Iteration, Attempt, nested planning depth rules, retry, deferral, child-workflow handoff (start/apply/cancel_child_workflow), execution outcome projections, or store protocol methods change.

Attempt Harness

Refresh when planner submission schemas, plan DAG semantics (the generator + reducer DAG and its structural gate), reducer launch, launch failure, exhaustion, or audit event behavior changes.

Context Engine

Refresh when role context shapes, XML tags, task guidance, skills, terminal catalogs, or launch message row rules change.

Agent Roles

Refresh when profile frontmatter, allowed tools, terminal sets, notification triggers, main role contract, helper profiles, or validation changes.

Terminal Tools

Refresh when submission tool names, schemas, terminal descriptors, advisor review focus, output validation, or terminal exclusivity changes.

Cross-Module Bridges

Refresh when Workflow entry bootstrap, sandbox binding, run_ephemeral_agent, ExecutionMetadata, message artifacts, or sandbox API routing changes.

Stale Claims #

  • docs/task_center_harness_and_context_engine.html remains historical reference material, not the canonical current wiki. Use this module for refreshed source-backed Workflow claims.
  • Current state-service naming is WorkflowLifecycle plus IterationAttemptCoordinator, not GoalHandler, IterationManager, IterationFactory, or IterationClosureRouter.
  • Current child-workflow handoff resumes the parent generator via AttemptOrchestrator.apply_child_workflow_outcome(), completing the parent task DONE or FAILED. There is no WorkflowClosureReportRouter / *ClosureReport / WorkflowOrigin abstraction and no final_outcome; do not document it as returning the parent to RUNNING.
  • The EVALUATOR role is gone - judgement is a reducer plan task using submit_reducer_outcome(status, outcome); the verifier profile and the evaluator context recipe were removed. The five roles are planner, generator (executor), reducer, helper (advisor), and subagent (explorer).
  • An attempt is a planner-authored plan: a DAG of generator + reducer tasks (edges are needs), and stages are PLAN -> RUN -> CLOSED (no GENERATE/EVALUATE). Do not repeat plan_spec, evaluation_criteria, or separate generator/evaluator stage claims.
  • Nested generators still see submit_workflow_handoff, but DisallowNestedWorkflowHandoff rejects the call at submission time. Do not repeat older launch-routing claims, and do not cite the old submit_execution_handoff name.
  • The current helper agent uses direct launch prompt assembly through ask_advisor; the Context Engine is not involved in helper prompt composition.
