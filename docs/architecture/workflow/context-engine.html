Workflow context is now an explicit role-scoped document. ContextEngine reads persisted Workflow -> Iteration -> Attempt state and returns an AgentContext; XML and task guidance are pure renderings of that same typed object. The engine does not own lifecycle policy, retries, terminal submission handling, delegated-workflow routing, or provider execution.

Scope And Verdict #

  • ContextScope carries launch identity: role, workflow id, iteration id, attempt id, and optional task id.
  • ContextEngineDeps carries the stores. Builders read state through this bundle and do not reach into globals.
  • ContextEngine.build(...) dispatches by scope.role and returns AgentContext, not a generic packet or recipe block list.
  • AgentEntryComposer resolves the profile, renders context XML as one user row, renders task guidance as a second user row, and appends the optional skill row.
Boundary Keep lifecycle decisions in Workflow orchestrators and coordinators. Context construction only projects the state the launched role can act on.

Launch Workflow #

[AgentLaunchFactory]
        | creates ContextScope.for_planner/generator/reducer
        v
[ContextEngine]
        | AgentContext
        +--> render_context_xml(context)
        +--> render_task_guidance(context)
        v
[AgentEntryComposer]
        | context row, guidance row, optional skill row
        v
[EphemeralAttemptAgentLauncher]
scopePlanner launches require workflow, iteration, and attempt ids. Generator and reducer launches also require task id.
terminal policyThe composer uses the profile terminal list directly. Nested planner deferral is a call-time prehook plus launch-time reminder notification; workflow delegation controls are ordinary non-terminal tools.
rowsContext XML and task guidance remain separate internal rows after the system prompt. Provider adapters may later join rows for transport, but Workflow keeps them distinct.

Role Context Shapes #

RoleContextHidden on purpose
planner<context role="planner"><workflow> containing workflow goal, prior iteration task outcomes, current iteration goal, and previous attempt execution outcomes.Planner outcomes, raw task transcripts, and internal attempt history for prior iterations.
generator<dependencies> with dependency task outcomes, then <assigned_task task_id="...">.Workflow goal, attempt-wide plan, reducer outcomes that are not dependencies, and planner outcome metadata.
reducerThe same dependency + <assigned_task> shape as generator. Reducer no longer receives <assigned_prompt>.Deferred-goal text and unrelated generator work outside declared dependencies.
 
<context role="generator|reducer">
  <dependencies>
    <dependency task_id="...">
      <task task_id="..." role="generator|reducer" status="success|failed">
        outcome text
      </task>
    </dependency>
  </dependencies>

  <assigned_task task_id="...">
    role-local task text
  </assigned_task>
</context>

Guidance And Wire Shape #

render_task_guidance(context) derives the role outline, context limits, and directive from AgentContext. There is no separate tag dictionary, outline walker, or token-budget priority system in the current context path.

Planner directive
Plan generator and reducer tasks for <current_iteration><goal>.
Generator directive
Complete <assigned_task> using <dependencies>.
Reducer directive
Complete <assigned_task> using <dependencies>.
Terminal catalog
skill_message._wrap_task_guidance() appends the profile's terminal-selection block. Generator terminal is submit_generator_outcome; delegated workflow controls are non-terminal profile tools.

Outcome Inputs #

Context renders execution evidence only. A task outcome is bounded to one task. Aggregate attempt, iteration, and workflow outcomes are projections over generator/reducer ExecutionTaskOutcome records.

planner task outcomesPlanner submissions mark the planner task complete and create generator/reducer Task rows. Planner outcome records are not part of attempt, iteration, workflow, or planner-history evidence.
attempt outcomesAttempt close persists generator/reducer execution outcomes from its task rows.
iteration outcomesIteration close projects successful reducer outcomes from all attempts plus failed generator/reducer outcomes from the final failed attempt.
workflow outcomesWorkflow outcomes are derived from the latest iteration outcomes.
delegated workflow outcomesDelegated workflow results are inspected by the launching agent through workflow tools and synthesized into that agent's own terminal outcome. ContextEngine does not copy delegated workflow outcomes into a waiting parent task.
