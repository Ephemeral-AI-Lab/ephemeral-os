The test suites that exercise the harness live alongside the runner under backend/src/task_center_runner/tests/. They split into mocked-agent tests (fast, deterministic, hit the real sandbox surface) and real-agent tests (slow, drive a live LLM through the same harness). Both lanes share a SWE-EVO image-backed workspace fixture.

Scope #

  • The shared conftest.py at tests/ only re-exports fixtures via pytest_plugins = ["task_center_runner.core.fixtures"]. Suite-specific behavior lives in sub-conftests.
  • Live tests intentionally do not load the repo's .env. Defaults come from ephemeralos.yaml; overrides must be process env vars.
  • Out-of-tree consumers exist too — backend/tests/unit_test/test_task_center_runner/ and backend/tests/unit_test/test_benchmarks/ import the harness for narrow unit-test coverage.

tests/mockDrives run_scenario_on_sweevo_image or run_scenario against the persistent SWE-EVO container. Scenarios + probes emit deterministic turns through ScenarioLoopRunner, and the real query loop executes the tool calls.
tests/real_agentDrives the production agent loop through the SWE-EVO image and the central-config-selected model. Used for full e2e validation including coding-plan mode.
Out-of-tree unit testsbackend/tests/unit_test/test_task_center_runner/ has test_no_core_imports.py (the runner-agnostic guarantee on core/engine.py) and lifecycle aggregates. backend/tests/unit_test/test_benchmarks/ covers the sweevo evaluation and snapshot verifier logic in isolation.

Markers And Conftest Wiring #

mockEvery item resolved under tests/mock/ gets pytest.mark.mock via the suite conftest's pytest_collection_modifyitems.
sandbox_integrationItems under tests/mock/sandbox/ get this marker. Use it to select sandbox-focused runs: pytest -m "mock and sandbox_integration".
task_center_integrationItems under tests/mock/task_center/ get this marker. Use it to select pipeline-state-machine runs.
real_agentItems under tests/real_agent/ get this marker. Use it to gate slow live-LLM runs.
  • Marker assignment uses Path.is_relative_to on the resolved item path, so symlinked test trees still get the right markers.
  • Both conftests import the same fixtures from task_center_runner.environments.sweevo_image.fixtures: sweevo_image_instance, sweevo_image_sandbox, and workspace.
  • The top-level tests/conftest.py stays minimal — it only registers the task_center_runner.core.fixtures plugin. Suite-local conftests own the marker rules.

Workspace And Sandbox Fixtures #

sweevo_image_instance (session)
   |   reads EOS_SWEEVO_INSTANCE
   v
sweevo_image_sandbox  (session, async)
   |   1. _acquire_sweevo_session_lock(instance_id)   # fcntl flock
   |   2. bootstrap_sandbox_provider()
   |   3. require_sweevo_image_provider_healthy(...)
   |   4. provision/resume sandbox by deterministic name
   |   5. setup_sweevo_sandbox(install_lsp=True)
   |   yield {sandbox_id, sandbox, snapshot_name,
   |          repo_dir, reused_existing}
   |   6. _release_sweevo_session_lock
   v
workspace            (function-scoped)
       on first use after a session lock: no reset
       otherwise: reset_sweevo_workspace(sandbox_id, install_lsp=True)
       returns the session sandbox dict
  • The session-scoped sandbox is shared across every test in the session. The function-scoped workspace resets /testbed between tests so probes do not see prior-test side effects.
  • Reset uses reset_sweevo_workspace, which reads sandbox labels (sweevo_instance, project_dir) and rebuilds the public-tool base via api.build_workspace_base.
  • The session lock is reentrant within one process so two fixtures requesting the same sandbox do not deadlock. Cross-process flock makes parallel pytest sessions for the same instance serialize.

Contract Helpers #

Module-private helpers (underscored) carry assertion vocabulary shared across many test files. They consume the rich RunReport built by ScenarioLifecycle.

_project_build_contractsAssertions for complex-project-build probes — checks that the expected sequence of write/edit/shell/LSP results appeared and that build evidence files exist on disk.
_focused_scenario_contractsAssertions for focused pipeline scenarios — verifies graph_summary shape, role task counts, attempt sequence, and retained audit events.
_layer_stack_occ_overlay_assertionsAssertions for LayerStack / OCC / overlay invariants — depth thresholds, squash events, applied-edit counts, conflict reasons.

Tests typically follow this shape:

async def test_some_pipeline_path(workspace):
    report = await run_scenario_on_sweevo_image(
        SomeScenario(),
        instance=workspace["instance"],
        sandbox_id=workspace["sandbox_id"],
        audit_dir=...,
    )

    assert report.task_center_status == "done"
    assert report.passed_prompt_inspections
    assert report.passed_sandbox_checks
    assert_graph_summary_shape(report, [...])

Live-Config Gates #

database_configured()bool(get_central_config().database.url) — fixture-level prerequisite for tests that need the per-test store bundle.
live_e2e_heavy_enabled()runner.live_e2e.heavy_enabled — flag for skipping heavy concurrency / build / LSP probes when the runner is not configured for them.
live_e2e_capacity_enabled()runner.live_e2e.capacity_enabled — flag specifically for FullSystemCapacityMatrix and its pack-catalog walk.
real_agent_max_duration_s()runner.live_e2e.real_agent_max_duration_s — wall-clock cap that real-agent tests pass to RunConfig.max_duration_s.

Tests call these helpers from pytest.mark.skipif / pytest.mark.skip_if_false so a misconfigured central config cleanly skips rather than failing.
