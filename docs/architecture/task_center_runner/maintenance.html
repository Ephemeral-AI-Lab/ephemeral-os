All pages in this module share one baseline commit: d32994314. Refreshing the wiki is a delta-first task: git diff the baseline against HEAD for each page's evidence paths, then update only the sections whose claims drifted.

Refresh Plan #

  • Run git log --oneline d32994314..HEAD -- backend/src/task_center_runner first. If there are no commits, the wiki is current; skip.
  • For each changed file, find the matching page via the evidence paths in section metadata. Use git diff on those exact paths to scope the section refresh.
  • Update data-last-reviewed-commit on the section AND the page-level data-page-baseline-commit after refreshing claims. They should match unless the page intentionally mixes commits.
  • Validate with check_html_wiki.py on every touched page before committing.

Stale-Claim Watch #

Areas that historically drift between code and docs. Audit these first on any refresh pass.

Sweevo CLI locationCommit f915434df relocated the benchmark to task_center_runner.benchmarks.sweevo. Watch for old backend.benchmark_sweevo references or python -m benchmark_sweevo command lines.
RunReport vs PipelineReportMock-rich fields (launches, tool_calls, prompt_inspections, sandbox_checks) live ONLY on RunReport. If a page describes them on PipelineReport, that page is stale.
Scenario registry sizeThe catalog grows over time. Re-read scenarios/__init__.py::SCENARIO_REGISTRY on every refresh and update the catalog page if new lanes appear.
Probe modesNew probe-mode tokens are easy to add in scenario_adapter.py, probe_bridge.py, or PROBE_BUILDERS without updating the mocked-agents page. Grep for new action-token branches and reflect them in the probe library section.
Daytona vs Docker vocabularyWatch for new code that mentions Daytona's state on the sweevo benchmark path. Docker's status is the authoritative key; the benchmark is docker-first.
Sandbox label keysreset_sweevo_workspace reads sweevo_instance and project_dir. New labels or renames here can break workspace reset; verify any change against _provision.py::_sweevo_sandbox_labels.
Bootstrap-required agents_REQUIRED_AGENT_NAMES in core/bootstrap.py is the canonical list. If markdown profile naming or the registry contract changes, this set and the mocked-agents page need to be updated together.
