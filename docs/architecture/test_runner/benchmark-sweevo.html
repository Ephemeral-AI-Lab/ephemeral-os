SWE-EVO is one benchmark adapter on top of the generic harness. It loads a Hugging Face instance, provisions a persistent Docker sandbox at sweevo-<instance_id>, runs the real LLM agent stack through run_pipeline, materializes the active overlay onto /testbed, then runs FAIL_TO_PASS and PASS_TO_PASS pytest sets to compute resolved and fix_rate.

REMOVED ⚠️ The Python backend/src/test_runner documented here has been removed. The Rust replacement is specified in integration-test/SPEC.md (see also docs/plans/test_runner_rust_SPEC.md) but is not yet implemented; the evidence paths below describe the removed Python design for historical reference.

Scope #

  • The benchmark adapter lives under backend/src/test_runner/benchmarks/sweevo/. Three stages are owned by three files: setup.py, run.py, eval.py. pipeline.py wires them.
  • The sandbox model is docker-first and persistent per instance. Sandbox status uses Docker's status key.
  • The adapter never touches the agent loop directly. It assembles a RunConfig and hands it to core/engine.py::run_pipeline; the runner delegate wraps engine.api.run_ephemeral_agent.
  • Grading is done by SweevoLifecycle.after_run against the real pytest invocation inside the container; the lifecycle materializes the active overlay onto /testbed before evaluating.

logging_configure_logging sets level to ERROR then disables WARNING globally — keeps the verdict line the dominant signal on stderr.

Three-Stage Workflow #

run_benchmark_sweevo(args)
   |
   |  Stage 1 — setup
   v
preflight(args)
   - load_pr_description(instance_id, csv_path)        -> goal text
   - load_sweevo_instance(source, instance_id)         -> SWEEvoInstance
   - bootstrap_sandbox_provider()                       -> EOS_SANDBOX_PROVIDER
   - verify_sweevo_snapshot_exists(instance)            -> fail-fast on missing
   - resolve audit_dir + max_duration_s
   - return PreContext
   |
provision_sandbox(ctx)
   - name = sweevo-<instance_id>  (deterministic)
   - existing = _find_existing_sandbox_by_name(...)
   - if existing is None: _create_sandbox(...)
     else:                _resume_sandbox(...)
   - setup_sweevo_sandbox(instance, sandbox_id, repo_dir, install_lsp=True)
       * wait_for_sandbox_exec_ready (6 attempts)
       * git reset --hard / clean -fd / checkout base_commit
       * pip install -e .
       * rebuild sweevo workspace base via daemon
       * conditional plugin install (LSP)
   - return sandbox_id
   |
   |  Stage 2 — runtime wiring
   v
build_run_config(ctx, sandbox_id) -> RunConfig
   sandbox       = SweevoProvisioner(instance, sandbox_id, install_lsp=True)
   runner_factory= build_agent_delegate(repo_dir)          -> wraps run_ephemeral_agent
   lifecycle     = SweevoLifecycle(instance, repo_dir, aggregate_jsonl_path)
   bootstrap     = bootstrap_real_agent_runtime            -> provider + stores + agents
   audit_dir     = ctx.audit_dir
   run_label     = benchmark/sweevo/<instance_id>
   instance_id   = ctx.instance.instance_id
   max_duration  = ctx.max_duration_s
   extras        = {"runtime_config": RuntimeConfig(cwd=Path.cwd())}
   |
   |  Stage 3 — execution + grading
   v
run_pipeline(config)
   |
   v
SweevoLifecycle.after_run(ctx, report)
   if report.workflow_status == "done" and not timeout:
       apply_layerstack_to_repo(sandbox_id, repo_dir)   -> daemon RPC + .git check
       evaluate_sweevo_result(instance, result, sandbox_id, repo_dir)
            - extract combined patch (git add -A + diff HEAD)
            - ensure_sweevo_test_patch (chunked base64 stage + git apply)
            - _run_test_set_outcome(F2P)  -> passed / runnable / dropped
            - _run_test_set_outcome(P2P)  -> passed / runnable / dropped
            - fix_rate = F2P_passed / max(F2P_total, 1)
            - resolved = (F2P all passed) and (P2P unchanged)
   else:
       result.error = "timeout" | workflow_status | "unknown"
   atomic_write_json(run_dir/sweevo_result.json, asdict(result))
   if aggregate_jsonl_path: append one JSON line
   |
   v
format_verdict(report) -> one-line stdout + exit code (0 resolved, 1 else)

Stage 1: Preflight And Provision #

dataset loaderload_sweevo_dataset reads from a Parquet path, a cached HF Arrow file under HF_DATASETS_CACHE, or falls back to datasets.load_dataset. functools.lru_cache keeps the call cheap inside a session.
PR description loaderload_pr_description(instance_id, csv_path) is strict — missing CSV, missing row, and empty value all raise. The default CSV was the removed Python sweevo_gpt5_2025_08_07_pr_descriptions.csv fixture; SWEEVO_PR_DESCRIPTIONS_CSV overrides.
prompt builderbuild_sweevo_user_prompt wraps the PR description in the canonical SWE-EVO <Workspace Root> / <pr_description> envelope plus an explicit instruction not to touch test files. Mocked-agent tests reuse this builder via environments/sweevo_image/fixtures.py.
provider bootstrapbootstrap_sandbox_provider() respects EOS_SANDBOX_PROVIDER + central config. Docker is the provider.
snapshot verifyverify_sweevo_snapshot_exists(instance) raises SnapshotNotRegisteredError when the snapshot is missing. Pre-register via test_runner.benchmarks.sweevo._snapshot.register_sweevo_snapshot(instance) before invoking the CLI — fails fast instead of after a 30-minute LLM run.
provisionprovision_sandbox creates or resumes the container. Docker name uniqueness rejects parallel duplicates. Resume branches: running reuses, exited|created|paused starts via start_sandbox, anything else deletes and recreates.
setup_sweevo_sandboxSix exec-ready attempts, then git reset / clean / checkout to instance.base_commit, pip install -e ., daemon-side api.build_workspace_base with reset=true, optional LSP plugin install via ensure_installed + api.plugin.ensure.

Stage 2: RunConfig Assembly #

SweevoProvisionerVerify-only provisioner. provision(ctx) re-runs setup_sweevo_sandbox(instance, sandbox_id, repo_dir, install_lsp=True) so retry attempts get a clean checkout, then returns a SandboxLease(sandbox_id, metadata={instance_id, repo_dir}). release(lease) is a no-op — the caller owns container lifecycle.
build_agent_delegateReturns a runner_factory(ctx) that yields an AttemptAgentRunner-shaped coroutine. The runner wraps engine.api.run_ephemeral_agent, normalizes ExecutionMetadata (sets sandbox_id, agent_name, repo_root, exec_cwd), and forwards every other kwarg.
build_run_configAssembles the RunConfig with extras["runtime_config"] = RuntimeConfig(cwd=Path.cwd(), external_api_client=None), bootstrap=bootstrap_real_agent_runtime, run_label="benchmark/sweevo/<instance_id>", and the SweevoProvisioner / delegate / lifecycle.

Stage 3: Lifecycle, Materialize, Grade #

before_run / on_event / on_abortedNo-ops except on_aborted stashes the timeout reason for after-run reporting.
after_runBuilds SWEEvoResult from the report, applies the layerstack overlay to /testbed, runs the grader, atomically writes sweevo_result.json to the run dir, and appends a one-line JSON record to aggregate.jsonl when the aggregate path is configured.
apply_layerstack_to_repoCalls api.commit_to_workspace on the daemon, then runs a sandbox-side test -d <repo_dir>/.git probe to catch the class of bugs where an overlay opaque-dir marker shadows the repo's .git.
ensure_sweevo_test_patchApplies the SWE-EVO test patch via chunked base64 (because sandbox_api.raw_exec has no stdin and inline argv would overflow). Three states: APPLYABLE → apply, ALREADY_APPLIED → log + skip, anything else → log warning with truncated stderr.
F2P / P2P run_run_test_set_outcome stages test_ids JSON via _write_file_via_chunked_base64, writes a real-file pytest runner script (python -c would break multiprocessing-spawn-based test suites like Dask), and runs the file via conda activate testbed && cd /testbed && python <runner>. Bounded retry loop drops unfindable IDs when pytest exits with EXIT_CODE=4 + ERROR: not found: lines.
scoringfix_rate = F2P_passed / max(F2P_total, 1); resolved is all F2P pass AND no P2P break. Unfindable P2P IDs are treated as dataset artifacts (logged) and do not count as P2P_broken.
aggregate.jsonlOne line per run: instance_id, run_id, resolved, fix_rate, F2P / P2P counts, duration_s, task counts, status, error, sandbox_id, ISO UTC timestamp.
format_verdictOne-line stdout: benchmark_sweevo request_id=... status=... resolved=... fix_rate=... sandbox_id=... run_dir=.... Exit code 0 if resolved, else 1.

SWE-EVO Image Environment #

The image-backed environment is shared by mocked-agent and real-agent tests. It exposes three pytest fixtures and one helper.

sweevo_image_instance (session)Resolves EOS_SWEEVO_INSTANCE (default dask__dask_2023.3.2_2023.4.0) and selects the matching SWEEvoInstance via select_sweevo_instance.
sweevo_image_sandbox (session, async)Provisions the persistent sweevo container under a host-side file lock (.sweevo_runs/locks/sweevo-<slug>.lock) so parallel pytest sessions cannot rebind the same workspace mid-test. Calls require_sweevo_image_provider_healthy first — Docker uses the SWE-EVO image directly; failures skip cleanly.
workspace (function-scoped)Returns the session-shared sandbox after running reset_sweevo_workspace(sandbox_id, install_lsp=True) on subsequent uses so each test sees a fresh checkout of /testbed.
run_scenario_on_sweevo_imageHelper that builds the SWE-EVO user prompt with build_sweevo_user_prompt(instance, repo_dir) and forwards to core/runner.py::run_scenario. Same call shape as the generic runner but pre-fills repo_dir and instance_id.
  • The session lock is fcntl-based; lock files live under .sweevo_runs/locks/ and carry the holder's PID. Acquisition is reentrant per process (count-tracked).
  • Tier-0 health was loaded dynamically from the removed Python tier0_health tooling so the architecture-level harness did not depend on the legacy test tree being importable as a package.

Persistent Sandbox Model #

  • Naming is deterministic: _sweevo_sandbox_name(instance) = sweevo-<instance_id>, truncated to 63 characters with an 8-char suffix preserved so two long-prefix instances do not collide.
  • Back-to-back runs reuse the same container: provision sees an existing match by name, calls _resume_sandbox, which starts the container if needed and skips creation.
  • The CLI does NOT destroy the container on exit. Reclaim disk with docker container prune or selectively: docker ps -a --filter 'name=sweevo-' --format '{{.Names}}' | xargs -r docker rm -f.
  • Sandbox status uses Docker's status key. "Dead", "removing", and "restarting" containers are detected and recreated.
  • Labels carry purpose=sweevo-test, project_dir, sweevo_instance, and sweevo_repo so reset_sweevo_workspace can resume a reused sandbox without an explicit instance argument.

Models And Constants #

SWEEvoInstanceOne benchmark row: instance_id, repo, base_commit, problem_statement, patch, fail_to_pass, pass_to_pass, docker_image, test_cmds, environment_setup_commit, test_patch, start_version, end_version, instance_id_swe, pr_description.
SWEEvoResultOne run's score: plan_id, instance_id, status ("pending" / "completed" / "failed"), agent_patch, resolved, fix_rate, F2P passed/total, P2P broken/total, duration_s, task counts, error, task_summaries.
PreContextOutput of preflight: instance, repo_dir, snapshot_name, goal, audit_dir, max_duration_s.

Timeouts

_DEFAULT_SANDBOX_COMMAND_TIMEOUT = 240s. _DEFAULT_SNAPSHOT_CREATE_TIMEOUT = 1200s. _DEFAULT_SANDBOX_SETUP_TIMEOUT = 360s. _DEFAULT_SWEEVO_TEST_TIMEOUT = 1200s. All multiplied by _REQUEST_TIMEOUT_MULTIPLIER = 2.

Conda activate

_CONDA_ACTIVATE = ". /opt/miniconda3/etc/profile.d/conda.sh && conda activate testbed". Prepended to every test-set command so SWE-EVO Docker images run pytest under the right environment.

Repo dir

_REPO_DIR = "/testbed". SWE-EVO Docker images mount the project there.

Dataset

_DEFAULT_DATASET_SOURCE = "Fsoft-AIC/SWE-EVO". _DEFAULT_SWEEVO_INSTANCE_ID = "dask__dask_2023.3.2_2023.4.0".

Sizing

count_sweevo_changelog_items counts bullet/numbered lines in problem_statement; classify_sweevo_instance_size labels them small (<5), medium (5–19), or large (≥20). select_sweevo_instance uses these to pick a representative instance when no explicit id is provided.

Snapshot policy

_has_explicit_sweevo_image_version rejects bare repos and :latest. Bare image refs skip the snapshot path entirely; explicit image@digest or image:<version> refs go through register_sweevo_snapshot + verify_sweevo_snapshot_exists.
