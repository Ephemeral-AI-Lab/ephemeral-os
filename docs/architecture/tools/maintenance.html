The HTML checker covers missing section baselines, broken local page links, unresolved placeholders, repeated ids, and missing page baseline markers.

Baseline Refresh Rule

  1. Run git diff <baseline>..HEAD -- <evidence paths> before changing a page claim.
  2. Update only the sections whose evidence paths changed or whose claim was proven stale.
  3. Move the section data-last-reviewed-commit to the reviewed commit after verification.
  4. Keep runtime refactoring out of wiki refreshes unless a separate user request explicitly asks for it.
