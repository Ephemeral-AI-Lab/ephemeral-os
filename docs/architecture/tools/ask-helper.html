ask_advisor and ask_resolver synchronously launch helper agents and return their terminal output to the caller.

Tool Shape #

backend/src/tools/ask_helper exposes ask_advisor and ask_resolver. They are blocking helper launches, not background tasks: neither tool declares background="always" or an optional background mode, and each awaits run_ephemeral_agent() directly inside its tool body.

ToolIntentHelper terminalUse case
ask_advisorREAD_ONLYsubmit_advisor_feedbackAudit a pending parent terminal submission before the parent submits it.
ask_resolverWRITE_ALLOWEDsubmit_resolver_resultFix concrete verifier or evaluator issues and report remaining gaps.

The helper tools reconstruct the helper launch messages directly instead of using the TaskCenter context composer. build_helper_messages() resolves the helper agent definition, reads the parent conversation from tool context, extracts the parent's first two user messages, resolves the parent agent definition when possible, captures active terminals, and builds a filtered transcript starting after the parent launch messages.

assemble_user_msg_1() wraps the parent context, parent task, and parent transcript as evidence with a prompt-injection guard. The second helper message is tool-specific: advisor gets terminal catalog plus pending submission; resolver gets issue list plus optional issue context.

If the runtime config, helper agent definition, or parent transcript shape is missing, the tool returns an explicit error ToolResult rather than launching a weakly grounded helper.

Advisor Flow #

  1. The parent supplies the terminal tool it intends to call and the exact payload it intends to submit.
  2. ask_advisor renders the parent's terminal catalog through render_terminal_catalog(..., focus="advisor_review_focus"). If parent terminals cannot be resolved, it uses a fallback note and still runs the helper.
  3. The pending submission is rendered as JSON so the helper can compare tool selection and payload support against the parent transcript.
  4. The helper is launched with role="advisor", agent_type="agent", persist_agent_run=False, and the current sandbox id when present.
  5. The helper must terminate with submit_advisor_feedback. The parent receives the terminal output and metadata, including the advisor verdict.

Resolver Flow #

  1. The parent supplies a non-empty list of concrete issues and optional context.
  2. ask_resolver builds a resolver task message with the issue list and instructs the helper to edit files as needed.
  3. The helper is launched with role="resolver", agent_type="agent", persist_agent_run=False, and the current sandbox id when present.
  4. The resolver uses ordinary tool registry permissions for file edits and sandbox work. Those calls still pass through the engine tool framework and sandbox daemon routing; the ask-helper wrapper does not create a custom workspace pipeline.
  5. The helper must terminate with submit_resolver_result. The parent receives the terminal output and metadata, including resolution status and any remaining issues.

Connections #

engineHelpers launch through engine.api.run_ephemeral_agent and block until the helper run returns.
terminal toolsHelpers terminate through their own submission terminals, then the ask-helper tool forwards that terminal output to the parent as the tool result.
TaskCenterHelper advice or fixes are evidence for the parent. They do not directly close the parent TaskCenter attempt or bypass the parent's terminal submission contract.
sandboxResolver edits and advisor reads use normal tool permissions and sandbox routing. Active sandbox id is forwarded, but background task supervision is not involved.
subagent contrastrun_subagent is always-background and returns through check/wait. Ask helpers are synchronous tool calls whose child terminal output is returned immediately to the current tool result.

Verification #

Tests cover helper retry behavior through run_ephemeral_agent, missing terminal-result errors, transcript/message composition, terminal catalog rendering, and advisor/resolver submission metadata. Refresh this page when helper prompts, transcript filtering, helper terminal contracts, or direct-launch runtime metadata change.
