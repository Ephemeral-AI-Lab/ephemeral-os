Subagent Tool

run_subagent is the always-background tool for launching focused worker agents without creating peer-to-peer orchestration.

Registration #

backend/src/tools/subagent registers one caller-aware tool: run_subagent. The framework factory uses make_subagent_tool_from_context(), which wraps the base tool in RestrictedRunSubagentTool and narrows agent_name to registered dispatchable subagents.

The underlying tool is declared with intent=Intent.WRITE_ALLOWED, background="always", and task_type=SUBAGENT_TASK_TYPE. That declaration is why the engine defers it from foreground streaming and gives the background supervisor a subagent-specific cancellation path.

Launch Workflow #

Parent callrun_subagent(agent_name, prompt)
Background dispatcheralways-background, alias bg_N
Validation gatesknown target, subagent type, no recursive spawn
Ephemeral child runstatic guidance plus caller prompt
Progress providerparent can peek child messages
Child terminalterminal output becomes parent background result
Parent resumeschecks result, continues, or submits TaskCenter terminal
  1. The parent calls run_subagent(agent_name, prompt). Engine background dispatch strips no user payload fields except any generic background flag, validates the input, assigns bg_N, and starts the tool body as a supervised asyncio task.
  2. The tool rejects missing runtime config, blank prompts, unknown targets, targets that are not agent_type="subagent", and recursive calls from a caller whose metadata says agent_type="subagent".
  3. The tool imports run_ephemeral_agent through engine.api, builds subagent metadata with agent_type="subagent" and role information, and forwards the active sandbox id.
  4. The child does not inherit a TaskCenter context packet from the parent. It receives the parent's free-text prompt as its initial user message and a static subagent guidance prompt as the run prompt.
  5. persist_agent_run=False keeps the helper run ephemeral. The parent remains responsible for final TaskCenter submission through its own terminal tools.

Progress and Results #

When the child agent is spawned, run_subagent registers a supervisor progress provider for the current background_task_id. The provider snapshots the child agent's latest conversation messages and renders text, thinking, tool-use, and tool-result blocks for check_background_task_result.

The child must call one of its registered terminal tools. run_ephemeral_agent returns an EphemeralRunResult; run_subagent forwards terminal_result.output, is_error, and terminal metadata to the parent, adding subagent_terminal_called=true. If the child crashes or exits without a terminal result, the metadata marks subagent_terminal_called=false, and the background check tool reports the task as failed with the latest peek instead of pretending the findings were delivered.

Cancellation is also subagent-aware. cancel_background_task asks the supervisor to request an early stop, cancel the child task, and allow the child path to salvage a partial result before it settles.

TaskCenter Boundary #

run_subagent is a worker launch tool, not a global orchestrator. It lets the parent ask for focused exploration or implementation help while TaskCenter state still advances through persisted terminal submissions, context packets, lifecycle reports, and attempt state machines.

The parent receives the subagent's terminal output as a normal background task result. The parent then decides whether to continue, call another tool, ask a helper, or submit a TaskCenter terminal tool. This keeps task handoff and completion state in TaskCenter rather than creating direct agent-to-agent control flow.

Sandbox behavior follows the child run's ordinary tool execution. If the child uses sandbox tools, those calls still route through sandbox APIs, the daemon workspace dispatcher, the ephemeral workspace overlay pipeline, direct LayerStack/OCC fast paths, or isolated workspace lifecycle according to the active workspace mode.

Verification #

Key tests cover factory policy, always-background exposure, retry behavior through run_ephemeral_agent, terminal metadata projection, failed/no-terminal outcomes, concurrent subagent calls, background status rendering, and event-printer display. Refresh this page when subagent target registration, terminal metadata, progress-provider rendering, or background cancellation semantics change.
