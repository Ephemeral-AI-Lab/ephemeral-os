## Coverage Gaps Found

- The 3.6 plan cases did not exist before this iteration.
- Project-build contracts checked completion and functional behavior, but not lowerdir O(1), upperdir size, manifest depth, read-only search publish behavior, or LSP refresh-vs-restart timing.
- Existing LSP timing reported `remount_count_delta=0` on samples where totals increased because counters were captured after `get_session()` reconciled the session.
- Overlay grep accepted directory roots only; file roots such as `/ephemeral-os/conftest.py` produced no candidates.

## Fixes

- `grep_compute` now treats a regular file root as a single candidate while retaining the existing no-follow directory walk for directories.
- LSP server timing now snapshots cached session counters before `get_session()` and attaches deltas after the operation.
- The shell/edit/LSP 3.6 assertion now separates:
  - refresh/remount samples: must show `remount_count_delta > 0` and `start_count_delta == 0`;
  - no-refresh warm-service samples: must stay under the 500 ms read-only plugin budget.
- The shell/edit/LSP probe emits an immediate second hover after the refresh-triggering hover so hover also has no-refresh warm-service samples.
