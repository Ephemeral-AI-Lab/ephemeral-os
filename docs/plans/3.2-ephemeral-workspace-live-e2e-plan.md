# 3.2 Live E2E Plan - Ephemeral Workspace

## Goal

Create the missing TaskCenter runner live E2E tier for default ephemeral workspace behavior. In-workspace `read_file`, `write_file`, and `edit_file` use the direct layer-stack/OCC fast path; shell/search and outside-workspace file operations use the per-call overlay path, capture changes when applicable, publish through OCC, and destroy upperdir/workdir state after each call.

## Performance Evaluation Contract

Every case in this folder is performance-evaluation focused, not just correctness focused.

Required assertions:

- Lowerdir disk is O(1) for N operations: direct file calls and private namespace calls report `resource.command_exec.workspace_tree_exists == 0` and `resource.command_exec.workspace_tree_bytes == 0` for every resource snapshot.
- Per-call overlay disk scales only with changed files: `upperdir_tree_bytes` may scale with mutation bytes, but `run_dir_tree_bytes` and scratch usage must return near baseline after each overlay call. Direct file calls must not create overlay run dirs.
- Latency stays low enough for agent use: read/grep/glob p95 should stay below 500ms once warm, write/edit p95 below 1s for small files, shell mount p95 below 5s under concurrency.
- OCC queue time is reported separately from mount/run/capture time so slow writes can be attributed to `occ.apply.*` rather than overlay mount.

## 3.2.x Complex Cases To Add

### 3.2.1 `test_ephemeral_all_verbs_publish_and_cleanup.py`

Scenario:

1. Use default mode only.
2. Run `write_file`, `read_file`, `edit_file`, `grep`, `glob`, and `shell` against a mini Python package under `/testbed/eph_case`.
3. After every call, inspect daemon runtime dirs and layer-stack manifest version.

Assertions:

- READ_ONLY calls do not publish.
- WRITE_ALLOWED calls publish exactly their changed paths.
- Direct file calls do not create overlay run dirs; overlay-backed calls remove every per-call `upperdir` and `workdir` after the call.
- `grep`/`glob` results include the writes committed by prior calls.
- `changed_paths` kinds distinguish write, delete, symlink, and opaque dir cases.

### 3.2.2 `test_ephemeral_concurrent_disjoint_writes_coalesce.py`

Scenario:

1. Launch eight concurrent `write_file` calls to disjoint paths.
2. Launch two concurrent shell calls that each touch one disjoint path.
3. Read the resulting files.

Assertions:

- Typed writes preserve `source="api_write"` and can disjoint-batch/coalesce.
- Shell captures preserve `source="overlay_capture"` and do not incorrectly coalesce across multi-path captures.
- All disjoint typed writes land.
- No upperdir remains after completion.

### 3.2.3 `test_ephemeral_same_path_conflict_and_retry.py`

Scenario:

1. Launch four concurrent writes to `/testbed/eph_case/shared.txt`.
2. Retry failed writes with fresh reads between attempts.

Assertions:

- Exactly one first-wave write succeeds or the results are typed OCC conflicts.
- Retried writes use fresh base versions and eventually commit.
- Final file content matches the last successful retry.
- No `internal_error` appears in `sandbox_events.jsonl`.

### 3.2.4 `test_ephemeral_outside_workspace_policy.py`

Scenario:

1. Read `/etc/hosts`.
2. Write `/tmp/eph-scratch.txt`.
3. Attempt to write `/etc/hosts`, `/proc/sysrq-trigger`, `/sys/kernel/printk`, and `/boot/grub.cfg`.

Assertions:

- Safe pass-through reads work.
- Allowed `/tmp` writes do not enter OCC workspace changed paths.
- Denylisted host writes fail before kernel execution with typed `forbidden_host_path`.
- In-workspace file paths use the direct fast path; outside-workspace paths fall back to the overlay policy path instead of forcing direct OCC projection.

### 3.2.5 `test_ephemeral_cancellation_drops_partial_upperdir.py`

Scenario:

1. Start a long shell writing a large file under `/testbed/eph_case/partial.bin`.
2. Cancel it before completion.
3. Run a normal read/write cycle.

Assertions:

- Cancelled shell does not publish partial file content.
- Lease and run dirs are cleaned up.
- Subsequent foreground operations are healthy.
- Cancellation evidence appears in `sandbox_events.jsonl` with the invocation id.

### 3.2.6 `test_ephemeral_lowerdir_disk_is_o1_under_100_calls.py`

Scenario:

1. Run 100 sequential small write/edit/read calls.
2. Sample scratch/runtime disk usage after every 10 calls.

Assertions:

- Lowerdir disk usage is O(1) relative to call count.
- Runtime scratch usage returns near baseline after cleanup.
- Manifest version advances only for write/edit/shell mutation calls.
- Direct API p95 latency remains within the warm-run budget while N increases, without requiring overlay mount timings.
