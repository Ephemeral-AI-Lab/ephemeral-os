# 3.1 Live E2E Plan - Layer Stack, OCC, and Overlay

## Goal

Own the live E2E tests moved out of the sandbox root that directly stress the layer-stack/OCC/overlay path: auto-squash commit resume, focused OCC conflicts, high-concurrency overlay writes, and heavy zoned shell I/O.

## Performance Evaluation Contract

These cases are the primary O(1) disk and low-latency pressure tests for the default workspace pipeline.

Required assertions:

- Lowerdir disk remains O(1) as operation count, worker count, and workspace size grow: every private namespace resource snapshot reports `resource.command_exec.workspace_tree_exists == 0` and `resource.command_exec.workspace_tree_bytes == 0`.
- `upperdir_tree_bytes` scales with changed file bytes, not repository size or number of concurrent readers.
- `run_dir_tree_bytes` and scratch filesystem usage are transient; no run dir remains after completed or cancelled invocations.
- Latency is attributed by family: `command_exec.mount_workspace_s`, `run_command_s`, `capture_upperdir_s`, `layer_stack.prepare_workspace_snapshot.total_s`, `occ.apply.*`, and `api.shell.total_s` must be present in `sandbox_events.jsonl` or `performance_report.json`.
- Synthetic conflicts must be typed conflicts, not `internal_error`.

## 3.1.x Complex Cases To Add

### 3.1.1 `test_high_concurrency_o1_disk_and_latency_budget.py`

Extend the existing high-concurrency scenario to assert:

- max `workspace_tree_bytes == 0`;
- max `workspace_tree_exists == 0`;
- write/edit p95 stays below the post-unify baseline budget;
- OCC queue wait is reported separately from mount time.

### 3.1.2 `test_heavy_io_upperdir_scales_with_payload_not_repo.py`

Extend the heavy I/O zoned scenario to compare:

- tracked-zone upperdir bytes against written payload size;
- outside-zone `/tmp` writes against workspace `changed_paths`;
- gitignored-zone writes against OCC changed path filtering.

The expected result is lowerdir O(1), upperdir proportional to mutation bytes, and no outside-zone path in OCC.

### 3.1.3 `test_auto_squash_does_not_materialize_workspace_tree.py`

Extend auto-squash commit-resume to assert:

- squash depth crosses threshold;
- commit-resume wait is present;
- no materialized workspace tree appears during the long mutation chain;
- final readback remains correct after squash.
