# 3.6 Live E2E Plan - Project Build

## Goal

Own the complex project-build scenarios moved out of the sandbox root. These tests exercise realistic mixed tool use over a multi-file project and must now report performance evidence, not only task completion.

## Performance Evaluation Contract

Project-build live E2E is the realistic sustained-workload complement to the synthetic layer-stack/OCC tests.

Required assertions:

- Lowerdir disk remains O(1) across hundreds or thousands of tool calls: `workspace_tree_bytes == 0` and `workspace_tree_exists == 0` in namespace-mode resource snapshots.
- Warm read/grep/glob/LSP latency stays low while edits accumulate. If latency rises, the report must distinguish LSP restart/remount, OCC queue wait, mount time, and command body time.
- Upperdir bytes scale with the current operation's changed files, not with total fixture size.
- Full-mode runs produce `performance_report.json` with per-tool p50/p95/max for `read_file`, `write_file`, `edit_file`, `shell`, `grep`, `glob`, and LSP calls.

## 3.6.x Complex Cases To Add

### 3.6.1 `test_project_build_full_o1_disk_budget.py`

Run the full project-build scenario and assert:

- max workspace tree bytes is zero;
- max upperdir bytes stays within the largest single-operation payload band;
- manifest depth remains within squash targets;
- no materialized lowerdir copy appears.

### 3.6.2 `test_project_build_grep_glob_low_latency_after_many_edits.py`

Run the grep/glob full scenario and assert:

- grep/glob p95 remains below the warm-run budget after many edits;
- search results reflect the latest committed layer-stack tip;
- read-only search calls do not publish or advance manifest version.

### 3.6.3 `test_project_build_shell_edit_lsp_remount_not_restart.py`

Run the shell/edit/LSP full scenario and assert:

- LSP diagnostics see shell and edit changes;
- `lsp.session.remount_count_delta` increases after writes;
- `lsp.session.start_count_delta` remains zero in the warm refresh path;
- LSP p95 remains below the read-only plugin budget.
