# 3.3 Live E2E Plan - Background Tool

## Goal

Verify the Phase 2.5 background contract through TaskCenter runner artifacts: background is an engine-owned task wrapper around the same daemon RPCs, cancellation is invocation-keyed, and overlay leases are cleaned up on natural exit, cancellation, timeout, heartbeat loss, and engine death.

## Performance Evaluation Contract

Every live case in this folder must collect `.sweevo_runs/scenario_logs/.../sandbox_events.jsonl` and `performance_report.json` when run through the scenario harness.

Required assertions:

- Lowerdir disk stays O(1) for N background operations: `resource.command_exec.workspace_tree_exists == 0` and `resource.command_exec.workspace_tree_bytes == 0` for private namespace overlay samples.
- Per-operation disk scales with the mutation payload only: `upperdir_tree_bytes` and `run_dir_tree_bytes` may grow during an invocation but must disappear after natural exit, cancellation, timeout, or engine death.
- Latency stays bounded while background work is active: foreground p95 `command_exec.mount_workspace_s < 5s`; cancellation recovery foreground call completes under 5s; dispatcher post-exhaustion read completes under 1s.
- Background cancellation never publishes partial workspace changes; `changed_paths` is empty for cancelled jobs and no stale lowerdir/missing-layer text appears in the sandbox event log.

## Existing Surface

Current tests in this folder already cover:

- `test_background_shell_golden.py`: natural completion for concurrent background shells.
- `test_background_shell_cancel.py`: cancel mid-flight and confirm foreground recovery.
- `test_background_shell_interleave.py`: foreground shell latency while a background lease is active.
- `test_background_shell_executor_exhaustion.py`: dispatcher survival under many cancelled launches.
- `test_background_shell_partial_write_cancel.py`: partial write cancellation discards upperdir.
- `test_background_shell_cancel_during_maintenance.py`: cancellation during maintenance window.
- `test_background_shell_late_cancel_race.py`: terminal status precedence near completion.
- `test_background_shell_engine_kill.py`: daemon cleanup when the engine dies.

## 3.3.x Complex Cases To Add

### 3.3.1 `test_background_mixed_fg_bg_same_path_conflict.py`

Scenario:

1. Launch a background shell that sleeps, then writes `/testbed/bg-shared.txt`.
2. While it sleeps, run a foreground `write_file` to the same path.
3. Let the background shell complete.

Assertions:

- Both calls use one `api.v1.shell` / typed tool envelope; no shell-job RPCs appear in `sandbox_events.jsonl`.
- Exactly one final content wins according to OCC rules.
- The loser reports a typed conflict or skipped publish, not `internal_error`.
- The foreground call p95 mount latency remains below 5 seconds.
- Lease release count equals launch count.

### 3.3.2 `test_background_heartbeat_loss_reaps_only_stale_bg.py`

Scenario:

1. Launch two background shell tasks for the same agent.
2. Continue heartbeat for task A and stop heartbeat for task B.
3. Run one foreground shell while task B is stale.

Assertions:

- Task B is cancelled by daemon TTL reaper.
- Task A completes normally.
- Foreground call is not cancelled.
- `sandbox_events.jsonl` contains heartbeat/reap evidence keyed by invocation id.
- No stale upperdir remains under the daemon runtime root.

### 3.3.3 `test_background_exit_iws_drains_agent_tasks.py`

Scenario:

1. Launch a long background shell for `agent-x` in default mode.
2. Attempt `enter_isolated_workspace` for `agent-x`.
3. Then open iws for another agent and launch a background task inside it.
4. Call `exit_isolated_workspace` while that iws background task is running.

Assertions:

- Enter for `agent-x` rejects while default background tasks are in flight.
- Exit for the iws agent calls the drain path before teardown.
- Drained task does not publish partial writes.
- Exit audit includes survivor count / drain timing.

### 3.3.4 `test_background_engine_restart_no_lease_leak.py`

Scenario:

1. Launch a long background shell that writes several chunks over time.
2. Kill the engine side before completion.
3. Let daemon TTL/orphan cleanup run.
4. Start a normal foreground shell and read/write cycle.

Assertions:

- The stale invocation is cancelled or reaped.
- No layer-stack lease, upperdir, workdir, or child process remains.
- Foreground operations after restart complete successfully.
- `.sweevo_runs/scenario_logs/.../sandbox_events.jsonl` has no `stale lowerdir`, `mount_failed`, or `manifest references missing layer` text.

### 3.3.5 `test_background_many_small_writes_do_not_starve_dispatcher.py`

Scenario:

1. Launch N background shells that each write a small distinct file and sleep briefly.
2. Interleave M foreground `read_file` and `write_file` calls.

Assertions:

- Dispatcher remains responsive: foreground p95 below 5 seconds.
- All completed background tasks publish their files.
- Cancelled or timed-out tasks publish nothing.
- Daemon in-flight count returns to zero.
- O(1) lowerdir fields remain zero for all N background operations.
