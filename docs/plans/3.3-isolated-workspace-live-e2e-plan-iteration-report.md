### Root-cause hypotheses and evidence

- Concurrency threshold failure:
  - Audit showed both tool calls completed within about 1.6 ms of each other and each recorded about 1.43 s `duration_s`; the fixed `<1.1s` wall threshold was too brittle for Docker Desktop overhead.
- Missing `tree-copy`:

- Typed read failures:
  - Current `read_file` primitive returns typed `content` and `exists`, not shell-shaped `stdout`.
- Pre-flight path failure:
  - `_lifecycle.py` no longer exists after the helper split; current lifecycle code is in `helper/lifecycle.py`.
- Error-kind failures:
  - Current production and `3.0` unit plan enumerate `already_open`, `not_open`, `quota_exceeded`, and `host_ram_pressure`.
- TTL active failure:
  - Audit showed `sandbox_isolated_workspace_evicted` before the long `tool_call` event completed. `run_in_handle` updated `last_activity` only after the subprocess returned, so TTL saw the handle as idle while work was active.
