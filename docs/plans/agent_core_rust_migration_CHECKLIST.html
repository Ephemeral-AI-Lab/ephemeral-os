Scope

In Scope

  • agent-core/ workspace behavior only: runtime, state, database stores, workflow lifecycle, engine loop, tools, provider clients, sandbox API/host facade, skills, agent profiles, plugin catalog, audit, and testkit.
  • Unit and behavior tests with mocked external edges: no live model provider, no live eosd, no Docker daemon requirement, and no network dependency.
  • Rust-native contracts: typed IDs, DTOs, store traits, terminal submissions, model-facing tool names, sandbox-facing caller_id, and root Task semantics.
  • Concrete Cargo gates that can run from agent-core/ in normal CI.

Out of Scope

  • The old bundled Python scenario in docs/plans/ultra_complex_bundled_scenario_CHECKLIST.html.
  • Live sandbox proof, LayerStack/OCC/overlay pressure tests, daemon protocol soak tests, and Docker image setup. Those belong to the root integration-test/SPEC.md or the sandbox/ workspace.
  • Any deleted backend/, pyproject.toml, uv, or Python test-runner implementation surface.
  • Adding a global orchestrator, peer-to-peer agent communication, or a synthetic root workflow to make tests easier.

Test Layers

agent-core test boundary

Layer 0: pure contracts
  types / config / state DTOs / DB repositories / schemas / provider projection

Layer A: engine + runtime
  ScriptedSource mocks the model
  FakeTransport mocks daemon RPC
  FakeProvisioner mocks sandbox provisioning
  real engine loop + real tool dispatch + real temp SQLite stores

Layer B: workflow
  QueueRunner or ScriptedRunner mocks the whole agent run
  real Workflow -> Iteration -> Attempt lifecycle and stores
  no engine loop, no LLM, no tools, no sandbox

Layer C: host facade, pure only
  mock ProviderAdapter or pure serializers verify Docker-only host behavior
  no live Docker in default agent-core tests

External: integration-test / sandbox workspaces
  live Docker + eosd + daemon + LayerStack/OCC/overlay proof

Layer A Rule

Root and delegated sub-agents reached through AppState are scripted with EventSource. Do not inject QueueRunner through AppState; that is not a real runtime seam.

Layer B Rule

Workflow tests construct AttemptDeps directly and park with QueueRunner when they need non-closure checkpoints such as RUN-stage rows before reducers launch.

Testkit Rule

eos-testkit owns the shared scripted LLM, fake daemon transport, metadata fixture, agent builders, and stream stepper. Runtime and workflow owned-type doubles stay crate-local under tests/.
