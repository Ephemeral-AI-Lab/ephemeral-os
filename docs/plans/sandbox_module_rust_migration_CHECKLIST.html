Current Scope

This page is for the new Rust sandbox migration only. It covers the sandbox-facing agent-core crates plus the sandbox workspace crates that own daemon transport, workspace routing, LayerStack, OCC, overlay execution, command sessions, isolated workspaces, plugin PPC/package setup, audit, config, and live Docker E2E.

A check is complete only when it is tied to a current Rust source path and either a focused Cargo test, a live E2E test, or an explicitly named manual evidence artifact.

Deliberate De-scope

  • No legacy backend/, pyproject.toml, uv, or Python tool/runtime surface.
  • No TaskCenter workflow DAG, planner, reducer, notification, advisor, subagent, or LLM scenario checks.
  • No old PTY public names such as pty_session_id or api.v1.pty.*.
  • No public isolated_workspace_id routing parameter; isolated mode routes through the active sandbox caller_id handle.
  • No Daytona or non-Docker Rust sandbox provider branch.

Acceptance Criteria

  1. G0 source gates pass in the owning workspace or each failure is named as pre-existing/concurrent with exact file or test evidence.
  2. xtask package produces the expected sandbox/dist/eosd-linux-amd64 artifact and metadata before live Docker closure.
  3. The live E2E suites for core, daemon, ephemeral_workspace, isolated_workspace, layerstack, occ, plugin, and pressure pass for the sandbox claim being closed.
  4. Any intentionally skipped live lane is tracked as a concrete blocker, not marked source-closed.
  5. Final stale-symbol scans show no old Python/backend, PTY, agent_id, or legacy E2E config names in active sandbox-facing Rust surfaces.

Refresh Triggers

  • Daemon op registration changes: update source gates, core, daemon, and operation-name rows.
  • Workspace routing changes: update source gates plus ephemeral_workspace, isolated_workspace, occ, and layerstack.
  • Plugin package/PPC/LSP changes: update source gates plus plugin, daemon, and pressure plugin lanes.
  • Config or Docker harness changes: update gate ladder, environment note, and pressure workload assumptions.
  • Agent-core sandbox API/host changes: refresh host/API source gates and rerun scoped agent-core checks.
