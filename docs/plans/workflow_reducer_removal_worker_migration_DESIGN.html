Consolidated design state after the reducer→worker fold + the co-design decisions: full plan in TaskOutcome::Planner JSON (approach A), opaque store-minted task_id, and the AttemptExecutionTree as the attempt↔task index (no attempt_id on the task store). The workflow store is workflows · iterations · attempts (lifecycle authority). Agent runs (root / planner / worker) are task_runs rows; advisor / subagent are parented_runs. An attempt links to its planner + worker runs through its execution_tree, not through an attempt_id column on the runs.

Workflow / Iteration / Attempt (lifecycle)

pub struct Workflow {
    pub id: WorkflowId,
    pub request_id: RequestId,
    pub parent_task_id: TaskId,            // the task that launched the workflow (root task)
    pub parent_agent_run_id: AgentRunId,   // RENAMED from launched_by_agent_run_id
    pub tool_use_id: Option<ToolUseId>,
    pub workflow_goal: String,             // RENAMED from goal
    pub status: WorkflowStatus,
    pub iteration_ids: Vec<IterationId>,
    pub created_at / updated_at / closed_at,
}

pub struct Iteration {
    pub id: IterationId,
    pub workflow_id: WorkflowId,
    pub sequence_no: i64,
    pub creation_reason: IterationCreationReason,
    pub workflow_goal: String,             // ADD: denormalized copy of Workflow.workflow_goal (context locality)
    pub iteration_goal: String,            // this iteration's own goal (was column `goal`)
    pub attempt_budget: AttemptBudget,
    pub status: IterationStatus,
    pub attempt_ids: Vec<AttemptId>,
    pub deferred_goal_for_next_iteration: Option<DeferredGoal>,
    pub created_at / updated_at / closed_at,
}

pub struct Attempt {
    pub id: AttemptId,
    pub iteration_id: IterationId,
    pub workflow_id: WorkflowId,            // denormalized; derivable via iteration_id
    pub attempt_sequence_no: i64,
    pub plan_id: PlanId,                    // MINTED by the Attempt at creation
    pub execution_tree: AttemptExecutionTree, // the attempt↔task index (see below)
    pub state: AttemptState,                // Planning{started} | Running | Closed{closure}
    pub created_at / updated_at,
}

AttemptExecutionTree (replaces tasks.attempt_id + the old MaterializedPlan)

pub struct AttemptExecutionTree {
    pub plan_id: PlanId,
    pub planner_task_id: Option<TaskId>,    // bound when the planner is spawned
    pub nodes: Vec<ExecutionNode>,           // materialized on submit_plan_outcome
}
pub struct ExecutionNode {
    pub work_item_id: WorkItemId,
    pub needs: Vec<WorkItemId>,             // DAG edges (for scheduling)
    pub task_id: Option<TaskId>,             // bound when THIS work item's worker is spawned
}
The tree is the single place that answers "which tasks belong to this attempt" and "which task ran work item w2". Because the scheduler needs the work_item → task bindings anyway, the tree is not extra overhead — it makes tasks.attempt_id redundant.

Task, AgentRun & ParentedRun rows

pub struct Task {                 // PURE execution state — NO workflow lineage
    pub id: TaskId,
    pub request_id: RequestId,
    pub role: TaskRole,                    // Root | Planner | Worker

    pub status: TaskStatus,
    pub agent_name: Option<String>,
    pub task_outcome: Option<TaskOutcome>,   // full info incl is_pass when terminal

pub struct AgentRun {             // root / planner / worker — AgentType::Agent family

    pub request_id: RequestId,

4 · Outcome model — one JSON union per family

  // == root_outcome
  Root    { is_pass: bool, outcome: String },
  // == planner_outcome (FULL plan; no is_pass — a returned plan is success)
  Planner {
    plan_spec: String,
    work_items: Vec<WorkItemSpec>,
    deferred_goal_for_next_iteration: Option<DeferredGoal>,
  },

}
pub struct WorkItemSpec {
  id: WorkItemId, agent_name: AgentName,
  work_spec: String, needs: Vec<WorkItemId>,
}

ParentedOutcome (parented_runs rows)

#[serde(tag="kind")]
pub enum ParentedOutcome {
  Advisor  { verdict: AdvisorVerdict, outcome: String },
  Subagent { outcome: String },
}
pub enum AdvisorVerdict { Approve, Reject }
Two families, two row types, not merged. TaskOutcome is 1:1 with TaskRole; ParentedOutcome is 1:1 with ParentedAgentRunKind. Agent-run rows keep the raw terminal_payload and expose typed outcome mirrors beside it: AgentRun.task_outcome and ParentedRun.parented_outcome.

Outcome taxonomy — two layers

Layer 1 — per-run deliverable (the actual data an agent run produced; stored as terminal payload plus typed outcome mirror). Folded into the two enums:

Layer 2 — aggregation projections (READ-SIDE; computed from the runs, never stored). Recursive; status rolls up bottom-up. "returned" = the last attempt / iteration after which no further retry / continuation was spawned (the terminal one).

// PlannerOutcome ≡ TaskOutcome::Planner payload;  WorkerOutcome ≡ TaskOutcome::Worker payload.
pub struct AttemptOutcome {
    pub status: bool,                         // planner returned a plan AND every worker is_pass

}

    pub deferred_goal: Option<DeferredGoal>,  // = returned_attempt.planner_outcome.deferred_goal_for_next_iteration
    pub attempts: Vec<AttemptOutcome>,
}
pub struct WorkflowOutcome {
    pub status: bool,                         // = returned_iteration.status
    pub iterations: Vec<IterationOutcome>,
}

These three replace the old lifecycle disposition enums (IterationOutcome::Complete/Continue/Failed, WorkflowOutcome::Succeeded/Failed): "continue vs complete" is now just deferred_goal.is_some(). They are computed on read — never stored. The stored lifecycle status (TaskStatus · AttemptStatus · IterationStatus · WorkflowStatus) stays as the in-flight / scheduling authority; the projection status: bool is the terminal pass/fail roll-up of the returned child.
Consequence — single source for the deferred goal: IterationOutcome.deferred_goal is extracted from the planner_outcome, so TaskOutcome::Planner.deferred_goal_for_next_iteration is the only stored copy ⇒ DROP the iterations.deferred_goal_for_next_iteration column (derive it when creating the next iteration). Edge: worker_outcomes.all(..) is vacuously true on an empty list, so an attempt whose planner failed (no plan, no workers) must short-circuit to status=false.

5 · DB schema (final)

-- WORKFLOW STORE -------------------------------------------------
CREATE TABLE workflows (
  id                  TEXT PRIMARY KEY,
  request_id          TEXT NOT NULL REFERENCES requests(id) ON DELETE CASCADE,
  parent_task_id      TEXT NOT NULL,
  parent_agent_run_id TEXT NOT NULL,            -- ← launched_by_agent_run_id
  tool_use_id         TEXT,
  workflow_goal       TEXT NOT NULL,            -- ← goal
  status              TEXT NOT NULL,
  iteration_ids       TEXT NOT NULL DEFAULT '[]',
  created_at TEXT NOT NULL, updated_at TEXT NOT NULL, closed_at TEXT
  -- DROP outcomes (read-side projection)
);
CREATE INDEX ix_workflows_parent_agent_run_id ON workflows(parent_agent_run_id);

CREATE TABLE iterations (
  id              TEXT PRIMARY KEY,
  workflow_id     TEXT NOT NULL REFERENCES workflows(id) ON DELETE CASCADE,
  sequence_no     INTEGER NOT NULL,
  creation_reason TEXT NOT NULL,
  workflow_goal   TEXT NOT NULL,                -- NEW: denormalized copy
  iteration_goal  TEXT NOT NULL,                -- ← goal
  attempt_budget  INTEGER NOT NULL,
  status          TEXT NOT NULL,
  attempt_ids     TEXT NOT NULL DEFAULT '[]',
  created_at TEXT NOT NULL, updated_at TEXT NOT NULL, closed_at TEXT,
  UNIQUE (workflow_id, sequence_no)
  -- DROP outcomes; DROP deferred_goal_for_next_iteration (derived from the returned attempt's planner_outcome)
);

CREATE TABLE attempts (
  id                  TEXT PRIMARY KEY,
  iteration_id        TEXT NOT NULL REFERENCES iterations(id) ON DELETE CASCADE,
  workflow_id         TEXT NOT NULL,
  attempt_sequence_no INTEGER NOT NULL,
  stage               TEXT NOT NULL,            -- Plan | Run | Closed
  status              TEXT NOT NULL,
  plan_id             TEXT NOT NULL,            -- minted by the Attempt
  execution_tree      TEXT NOT NULL DEFAULT '{}', -- AttemptExecutionTree JSON (THE attempt↔task index)
  fail_reason         TEXT,
  created_at TEXT NOT NULL, updated_at TEXT NOT NULL, closed_at TEXT,
  UNIQUE (iteration_id, attempt_sequence_no)
  -- DROP planner_task_id (→ tree) · generator_task_ids · reducer_task_ids · work_item_ids · outcomes · deferred_goal
);

-- AGENT-RUN ROWS -------------------------------------------------
CREATE TABLE task_runs (                        -- root + planner + worker
  task_id          TEXT PRIMARY KEY,            -- OPAQUE, store-minted at spawn_agent
  agent_run_id     TEXT NOT NULL UNIQUE,        -- v4
  request_id       TEXT NOT NULL REFERENCES requests(id) ON DELETE CASCADE,
  role             TEXT NOT NULL,               -- 'root' | 'planner' | 'worker'
  status           TEXT NOT NULL,
  agent_name       TEXT NOT NULL,
  terminal_payload TEXT,                         -- raw terminal tool result
  task_outcome TEXT,                              -- typed mirror: TaskOutcome JSON
  token_count INTEGER NOT NULL DEFAULT 0, error TEXT,
  created_at TEXT NOT NULL, updated_at TEXT NOT NULL, finished_at TEXT
  -- DROP attempt_id · workflow_id · iteration_id · work_item_id (the attempt↔task link is attempts.execution_tree)
);

CREATE TABLE parented_runs (                    -- advisor + subagent
  task_id TEXT PRIMARY KEY, agent_run_id TEXT NOT NULL UNIQUE,
  request_id TEXT NOT NULL REFERENCES requests(id) ON DELETE CASCADE,
  status TEXT NOT NULL,
  parent_agent_run_id TEXT NOT NULL, parent_task_id TEXT NOT NULL,
  kind TEXT NOT NULL CHECK (kind IN ('subagent','advisor')),
  tool_use_id TEXT, agent_name TEXT NOT NULL,
  terminal_payload TEXT,                          -- raw terminal tool result
  parented_outcome TEXT,                          -- typed mirror: ParentedOutcome JSON
  token_count INTEGER NOT NULL DEFAULT 0, error TEXT,
  created_at TEXT NOT NULL, updated_at TEXT NOT NULL, finished_at TEXT
);
-- `tasks` (schedulable) mirrors task_runs: id, request_id, role, instruction, status, agent_name,
--  task_outcome — also with NO lineage columns.

7 · Spawn & launch (the id-minting path)

pub enum SpawnAgentTarget {
  Root     { request_id: RequestId },
  Workflow { coords: WorkflowCoordinates, role: WorkflowTaskRole,
             plan_id: PlanId, work_item_id: Option<WorkItemId> },  // Some = worker, None = planner
  Subagent { parent: ParentRef },
  Advisor  { parent: ParentRef },
}
// spawn_agent(target): agent_run_id = v4; task store mints the OPAQUE task_id; creates the run row.

pub struct AgentLaunch {        // ephemeral; one struct + kind (was 3 structs + ~11 match accessors)
  task_id, request_id, coords: WorkflowCoordinates,
  agent_name, context, task_guidance, agent_def, skill,
  kind: AgentLaunchKind,
}
pub enum AgentLaunchKind { Planner { plan_id: PlanId }, Worker { work_item_id: WorkItemId } }
The lineage (WorkflowCoordinates = workflow_id + iteration_id + attempt_id) rides on the ephemeral launch / spawn target (needed for the record path), never on the persisted Task. The persisted attempt↔task link is the execution_tree.

Generated as a design snapshot. Lifecycle status is the authority for pass/fail at the workflow/iteration/attempt levels; is_pass on a root/worker outcome is the model-reported result that the terminal also maps onto TaskStatus.
