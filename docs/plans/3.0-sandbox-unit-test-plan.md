# 3.0 Unit Test Plan - Sandbox Workspace Unification

## Goal

Pin the low-level contracts that the TaskCenter runner live E2E tests depend on, without booting SWE-EVO containers. These tests should fail before any expensive live run when an implementation regresses overlay lifecycle, workspace routing, background cancellation, plugin intent dispatch, or LSP refresh wiring.

## Test Groups

### 3.0.1 Overlay and namespace primitives

Target files:

- `backend/tests/unit_test/test_sandbox/test_overlay/test_namespace_runner_cancellation.py`
- `backend/tests/unit_test/test_sandbox/test_overlay/test_upperdir_capture.py`
- `backend/tests/unit_test/test_sandbox/test_execution/test_overlay/test_kernel_mount.py`
- new `backend/tests/unit_test/test_sandbox/test_overlay/test_overlay_handle_contract.py`

Cases:

- `test_overlay_handle_release_is_idempotent`: construct an `OverlayHandle` with a release closure and explicit `run_dir`; call `release()` twice; assert lease release and `run_dir` cleanup happen once.
- `test_kernel_umount_lazy_raise_modes`: cover `umount(path, lazy=False/True, raise_on_failure=False/True)` including the LSP remount mode `lazy=True, raise_on_failure=True`.
- `test_namespace_runner_cancel_kills_child_and_discards_upperdir`: cancel an in-flight namespace run; assert child process cleanup, no capture, and lease release.
- `test_capture_changes_preserves_source_tags`: write/delete/symlink/opaque-dir conversions preserve `api_write` for typed verbs and `overlay_capture` for shell/multi-path capture.

### 3.0.2 Ephemeral pipeline routing

Target files:

- `backend/tests/unit_test/test_sandbox/test_workspace_unification_phase2.py`
- new `backend/tests/unit_test/test_sandbox/test_ephemeral_pipeline_unified_lifecycle.py`

Cases:

- `test_ephemeral_write_acquire_run_capture_commit_destroy_order`: fake layer stack + fake namespace runner; assert exact order and one lease release.
- `test_ephemeral_read_skips_commit_but_still_destroys`: READ_ONLY request mounts and runs but does not call OCC publish.
- `test_ephemeral_same_path_concurrent_conflict_is_typed`: two same-path writes against a stale base return one success and one typed conflict.
- `test_out_of_workspace_paths_use_same_pipeline`: `/etc/hosts` read and `/tmp/scratch` write go through the same request path; denylisted host writes are rejected before kernel execution.

### 3.0.3 Isolated pipeline and lifecycle

Target files:

- `backend/tests/unit_test/test_sandbox/test_isolated_workspace_lifecycle_background.py`
- new `backend/tests/unit_test/test_sandbox/test_isolated_pipeline_unified_lifecycle.py`
- new `backend/tests/unit_test/test_sandbox/test_isolated_workspace_no_publish.py`

Cases:

- `test_enter_exit_error_kinds`: assert `already_open`, `not_open`, `quota_exceeded`, and `host_ram_pressure` are reachable from production code paths.
- `test_iws_tool_call_never_publishes_occ`: isolated write captures changed paths for observability but never calls OCC publish.
- `test_same_session_tool_calls_do_not_share_per_call_lock`: concurrent tool calls inside one handle overlap at the pipeline boundary.
- `test_exit_drains_agent_background_tasks`: `exit_isolated_workspace` calls `BackgroundTaskManager.cancel_by_agent` before teardown and reports survivors.
- `test_no_freeze_contract_left`: production isolated workspace code has no `freezer_degraded` telemetry contract and no idle freeze/unfreeze control path.

### 3.0.4 Background lifecycle

Target files:

- `backend/tests/unit_test/test_sandbox/test_static_background_lifecycle.py`
- `backend/tests/unit_test/test_sandbox/test_api/test_shell_background_dispatch.py`
- new `backend/tests/unit_test/test_sandbox/test_inflight_invocation_registry.py`

Cases:

- `test_pipeline_body_has_no_background_branch`: AST check over `EphemeralPipeline.run_tool_call` and `IsolatedPipeline.run_tool_call`.
- `test_shell_background_uses_single_rpc_envelope`: background shell call still dispatches `api.v1.shell`; background metadata is `invocation_id` plus `background=true`, not shell launch/reap/poll/cancel RPCs.
- `test_wire_cancel_precedes_local_task_cancel`: engine cancellation sends `api.v1.cancel(invocation_id)` before local asyncio task cancellation.
- `test_heartbeat_reaper_cancels_stale_background_only`: stale background invocations are cancelled; foreground invocations are not TTL-eligible.

### 3.0.5 Plugin and LSP runtime

Target files:

- `backend/tests/unit_test/test_sandbox/test_plugin_overlay_dispatch.py`
- `backend/tests/unit_test/test_sandbox/test_plugin_session.py`
- `backend/tests/unit_test/test_sandbox/test_plugin_projection.py`
- new `backend/tests/unit_test/test_sandbox/test_plugin_intent_dispatch.py`
- new `backend/tests/unit_test/test_plugins/test_lsp_session_overlay_refresh.py`

Cases:

- `test_tool_decorator_requires_intent`: importing a tool without `intent=` raises.
- `test_read_only_plugin_does_not_allocate_operation_overlay`: READ_ONLY plugin tool uses its service and leaves lease count unchanged.
- `test_write_allowed_plugin_uses_overlay_and_occ`: WRITE_ALLOWED plugin tool allocates overlay, captures changes, publishes, and releases.
- `test_lifecycle_intent_rejected_for_plugin_tools`: plugin registration rejects `Intent.LIFECYCLE`.
- `test_lsp_subscribes_through_pipeline_api`: LSP session manager uses `subscribe_workspace_changes` and `unsubscribe_workspace_changes`, not direct `event_bus` access.
- `test_namespace_remount_failure_is_loud`: stuck unmount during LSP remount raises instead of silently serving stale diagnostics.
