Rust Parity Audit — Authoritative Synthesis Report

Scope: EphemeralOS Rust port (two workspaces: agent-core/ and sandbox/, joined by eos-protocol) vs Python/docs ground truth (docs/architecture + backend/src + materialized pre-cutover Python at /tmp/oldpy). This report synthesizes 25 independently-investigated-and-verified areas. This audit gates deletion of backend/src.

Source precedence everywhere: Python ground truth > architecture docs > checklist wording. Where the independent verifier disagreed with the investigator, this report prefers the verifier and marks the disagreement.

1. Executive summary

Two-layer verdict. Per-component fidelity is HIGH; production integration is INCOMPLETE. backend/src deletion is NOT yet safe.

  • **Component fidelity (the dynamics, in isolation) is high and frequently byte-exact.** The sandbox storage spine — overlay mount/capture, LayerStack lease/snapshot/publish, squash segmentation + deferred GC, OCC commit gate, CAS hashing, persistence/state stores — reproduces the Python dynamics faithfully, much of it re-derived bilaterally by the verifiers (constants like AUTO_SQUASH_MAX_DEPTH=100, MAX_OCC_CAS_RETRIES=3, batch=64/window=0.002, ceil(1.5×limit) ceiling, the present_status vs _normalize_status outcome split, the 6-tool terminal enum, the no-inflight gate fail-open tags). Several invariants are stronger in Rust (build-time no-OCC guard for isolated workspaces; atomic JSON-append; CAS-based root-fail; extra bridge peer-isolation nft rules).
  • But the integrated runtime cannot complete a real request in a default build. Three production seams are non-functional stubs, and they compound:
    1. Advisor gate is a deny-all stub (AdvisorService::approval_status returns approved:false, reason:"missing" for every tool). The only approving AdvisorPort is #[cfg(test)]. Because Rust also gates submit_root_outcome (a deliberate divergence from Python, which does not), every advisor-gated terminal — planner/generator/reducer and root — is permanently denied in a default build. (advisor D1/D2, tools_framework D1)
    2. The shipped binary builds an empty agent registry (no agents_dir, no injected registry, no config fallback), so root never resolves and every request fails at root resolution. The request→completion logic is proven only by injection-based integration tests. (request_completion NF1)
    3. The subagent supervisor is a hollow stub that never launches a child, never surfaces a result, and pins a Running record that — via the no-inflight terminal gate — practically wedges the parent's terminal. (subagent D1/D9, background_supervisor §4/§5/NF-3)
  • Headline real bugs / missed dynamics (most likely to bite, ranked):
    1. **Subagent supervisor is a non-functional stub + phantom Running record wedges the parent terminal** (CRITICAL; unflagged silent miss with an affirmative-but-false ports.rs claim).
    2. **Advisor pass-before-terminal gate decision logic is a deny-all stub; root additionally gated** → no gated terminal can pass in production (HIGH; flagged "engine-only phase").
    3. Background-task supervisor lifecycle is unwired in the query loop — no creation, no per-turn completion drain, no terminate_for_parent_exit on TOOL_STOP, no cancel_all on hard-ceiling exit (HIGH; one root cause behind query_engine D1, terminal_tools D2, background_supervisor §4/§5).
    4. Provider-message sanitization missing — Rust sends the raw durable transcript to the provider; orphaned tool_use/tool_result pairs can produce Anthropic 400s (HIGH; query_engine D2 = model_provider NF1).
    5. Termination-condition prompt rewritten — the one-way-exit WARNING block and <Termination Condition> wrapper are gone; the model receives materially different guardrail text (HIGH; model_provider D1).
    6. Continuation deferred-iteration start-failure has no compensation — a single planner-launch failure on a deferred handoff leaves the workflow permanently OPEN and leaks the coordinator (HIGH; workflow_lifecycle D5 = deferred_goal_depth inv-1).
    7. write_stdin wire op-name mismatch — client sends api.v1.exec_stdin, daemon registers only api.v1.write_stdin → unknown_op, write_stdin broken end-to-end against the real daemon; the host fail-closed retry set carries the same wrong string, so a fix that only renames the client re-opens a replay-double-apply hazard (HIGH; sandbox_tools D1/D2 = daemon_protocol D1).
    8. OCC gitignore routing is a hand-rolled root-only matcher — nested .gitignore, dir-only patterns (node_modules/), **, and *-not-crossing-/ are all wrong; mis-routing flips DIRECT↔GATED and can silently clobber a tracked file or spuriously conflict on a build artifact (HIGH; occ D1 + verifier N2/N3).
    9. Per-agent skill scoping dropped — any agent can read any skill's reference docs and the not-found error leaks all bundled skill names (HIGH; tools_framework D7).

Several of items 1–3 are explicitly the "engine-only / Phase-6/7" migration frontier. The audit's job is to record that this frontier is not yet crossed: the data-plane port is trustworthy, but the agent-loop integration (advisor runner, subagent runner, background dispatch, workflow agent runner, registry wiring) is incomplete, so the live system cannot yet run end-to-end without test-only injection. Delete backend/src only after these production seams land and an end-to-end (non-injected) run of root → delegate → terminal is demonstrated.

Flagged-deferral vs unflagged-silent-miss: advisor (and attempt_harness D5, RuntimeAgentRunner) self-document the gap ("engine-only phase" / "Phase-7"). The subagent supervisor and the empty-registry binary are the opposite — no marker, and ports.rs:202-204 affirmatively but falsely claims the implementor "validates … and supervises terminal-result delivery." Treat the latter as silent misses, not deferred-by-design.
