## Deliverables (Stage 1 only)

1. **`src/config.rs` — promote to the full orchestrator `RunConfig` + clap `Args`** (spec 360-382), without breaking the test side. The existing `RunConfig::from_env`/`Manifest` reader (consumed by `fixtures.rs:33`) is a *different concern* sharing the name — **rename it** (e.g. `ManifestConfig::from_env`) and add the orchestrator `RunConfig`: `run_id` (`--run-id` verbatim, else `r{ts}-{sha256(git_HEAD ‖ test_manifest_hash ‖ EOS_E2E_RUN_SALT)[..8]}` via `sha2`; `ts` pinnable by `EOS_E2E_RUN_CLOCK`; reject chars outside `[A-Za-z0-9._-]` at parse), `max_parallel` (`--max-parallel` > `EOS_E2E_MAX_PARALLEL` > `available_parallelism().min(8)`), `tests: TestSelection`, `image`, `run_root` (`${EOS_E2E_RUN_ROOT:-$TMPDIR/eos-e2e}/{run_id}`), `gateway_socket` (`--gateway-socket`, required — attach-only), `build: BuildSource`, `cli_timeout` (30s), `gateway_ready_timeout` (5s), `cleanup: CleanupPolicy`. Add enums `TestSelection { All, Names(Vec<String>), RerunFailedFrom(PathBuf) }`, `CleanupPolicy { Always, OnSuccess, Never }`, `BuildSource { Cargo { profile }, Prebuilt(PathBuf) }`. Precedence is flag > env > default. Add `clap` to `Cargo.toml` (`clap.workspace = true`); `sha2`/`time` likewise (workspace deps).

2. **`src/report.rs` — add the outcome DTOs + `result.json` + `summary.json` + `run-manifest.json` writers** (keep `write_exchange`). Per spec 626-664:
   - `TestOutcome { schema_version, test_name, sandbox_id, status (passed|failed|skipped|errored), assertions { total, failed }, duration_ms, failure? }` → `reports/{sandbox_id}/result.json`.
   - `summary.json`: `{ schema_version, run_id, git_head, started_at, finished_at, max_parallel, status, counts{total,passed,failed,skipped,errored}, tests[], failed_tests[], artifacts_root, timing{...}, cleanup{...} }`, built **solely by globbing `reports/*/result.json`** — a missing `result.json` is an `errored` test; **never parse libtest stdout**. `timing` carries `build{...}` + `runner{...}` + `per_test[]` sub-objects (spec 657-661). Every artifact carries `schema_version`.
   - `run-manifest.json` writer: `{ schema_version, git_head, config, gateway_socket, run_id, image, clock }`. ⚠️ Its field set **must stay readable by `ManifestConfig::from_env`** (`schema_version`, `gateway_socket`, `run_id`, `image` at minimum) — the tests read this exact file.

3. **`src/fixtures.rs` — emit `result.json` per executed test.** Extend the existing `Sandbox::drop` (which already flushes `exchange.jsonl`) to also write `result.json` under the same `reports/{id}/`: determine status via `std::thread::panicking()` (true ⇒ `failed`, else `passed`), record a per-test `duration_ms` (start an `Instant` at provision), and use the provision `slug` as `test_name`. Counting `assertions{total,failed}` is **best-effort**: if cheap, back it with a thread-local counter bumped by the `assertion::` helpers; otherwise emit `{total:0,failed:0}` and leave the precise count to a follow-up — do not over-build it. **Keep the skip path a bare `return` that writes nothing** (env-unset only; the orchestrator always sets `EOS_E2E_RUN_ROOT`, so under orchestration nothing skips and `counts.skipped` is 0).

4. **`src/cleanup.rs` (NEW, SRP) — the RAII run guard + sweep** (spec 666-700). On drop / explicit run: (a) sweep survivor sandbox ids — read them from the `reports/*/` directory names (ids are minted inside test processes, not the orchestrator) and issue `sandbox-cli manager destroy_sandbox --sandbox-id {id}` for each (idempotent; the per-test RAII drop already reaps the happy path, this catches survivors); (b) **detach only** — never stop a gateway the runner did not start (attach mode); (c) `remove_dir_all(run_root)` gated by `CleanupPolicy` (default `OnSuccess`: keep on failure for inspection, remove on success; `--keep-artifacts` forces keep). Record the result (ids/dirs removed) into `summary.cleanup`. Support a standalone `--clean-run {run_id}` that repeats teardown. Wire `pub mod cleanup;` into `lib.rs`.

5. **`src/bin/eos-e2e.rs` — the orchestrator** (replace the stub), per the data-flow diagram (spec 313-334):
   - Parse `RunConfig` (flags/env); allocate/validate `run_id`.
   - **Preflight** (spec 706-721; also expose as an `eos-e2e preflight` subcommand): Linux → `docker version` → `docker image inspect {image}` → real-runtime probe (a cheap `manager` call on `--gateway-socket`; if the response error contains "runtime is not configured", **exit 2** naming the missing real-runtime gateway). Any failure exits `2` with the exact missing item.
   - Write `run-manifest.json` (deliverable 2).
   - **Phase A build**: skipped in attach-only Stage 1 (`--gateway-socket` given ⇒ `BuildSource::Prebuilt`-equivalent ⇒ `timing.build.*_ms = 0`); `sandbox-cli` must be on `PATH`.
   - **Phase B**: `gateway::await_ready` (reuse), export `EOS_E2E_RUN_ROOT={run_root}`, run `cargo test -p sandbox-e2e-live-test --test manager -- --test-threads={max_parallel}`. **⚠️ This `--test manager` is the single stage-aware line** (phases-note 44-49, 70) — Stage 2 will drop the restriction for the full suite; isolate it so the flip is one edit. Honor `TestSelection` (libtest name filters; `RerunFailedFrom` parses `failed_tests[]` from a prior `summary.json`).
   - **Aggregate**: glob `reports/*/result.json` → `summary.json`. Pass/fail gate = the **`cargo test` process exit code**.
   - **Cleanup** per policy (deliverable 4).
   - Return `ExitCode::SUCCESS` iff `cargo test` exited 0 **and** every `result.json` is `passed`. On failure print the exact focused-rerun line (spec 811-818).

## Fences (do not cross)
- **Black-box only**: every sandbox/runtime op via `sandbox-cli` over the gateway socket; no internal-crate deps, no test-injected runtime. Cleanup destroys only **captured ids** (from `reports/*/` dir names) + `remove_dir_all(run_root)` — it must be provably this-run-only and can never touch a sibling run.
- **Stage fence**: default target is **manager-only** (`--test manager`). Do NOT author/enable any `tests/runtime/**` (R2-R8, N2 are Stage 2); leave the dormant R1 `one_shot.rs` untouched. Do not add a runtime-readiness skip probe — the gate is binary-level (phases-note 51-59).
- **Attach-only**: no gateway spawn, no `package-fast` binary discovery, no Docker run-label backstop (Open Items #1/#2 — out of scope). Cleanup detaches; it never stops a gateway.
- **Phase 4 is out of scope**: no `get_observability_tree` polling and no `observability.json` here.
- `build.rs` and `tests/{manager,runtime}.rs` are **unchanged** — adding nothing to the test tree.

## Conventions
SRP, one job per unit; prefer less (no field/struct/method an existing one already carries). **No inline comments in production code** (doc comments on public items OK; test-intent comments OK). Daemon-protocol vocabulary stays in `sandbox-protocol` — this crate stays behind the public socket via `serde_json::Value`. External deps via `dep.workspace = true` only (add `clap`, `sha2`, `time`, and `tokio`/`tokio-util` only if actually used — the orchestrator may stay sync). Clippy must pass: no new `unwrap_used`/`dbg_macro`/`undocumented_unsafe_blocks`; justify any `unsafe` with a `// SAFETY:` block.

## Acceptance (in order)
