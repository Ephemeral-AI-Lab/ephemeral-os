5. Component Legend

🟥 EphemeralAgent

Short-lived agent per request. Created via spawn_agent(). Dies after run(prompt) completes. Owns its own API client, tool registry, and QueryEngine.

🔵 QueryEngine

Owns conversation history (_messages) and cost tracker. Delegates to run_query() for the actual loop. Session state persists across ephemeral agents.

🟢 run_query() Loop

Core loop: stream from API → yield events → check tool_uses → execute tools (parallel if multiple) → append results → repeat until the model stops requesting tools, a submission tool completes, or the tool-call budget is exhausted.

🟨 Tool Execution

PRE_TOOL_USE hook → validate input → tool.execute() → POST_TOOL_USE hook → ToolResultBlock. Unknown/invalid tools return error results, not exceptions.

🟣 HookExecutor

Lifecycle events: PRE_TOOL_USE (can block), POST_TOOL_USE (observational). Enables permission checks, logging, monitoring.
